QuantStudio Design & Analysis Adapter Requirements

Revision 05

Date: 2024-04-19

Background Section

The QuantStudio Design & Analysis adapter is intended to convert .xlsx exports from ThermoFisher Scientific’s QuantStudio Design & Analysis software and generate a .json file corresponding to the qPCR Allotrope Simple Model. QuantStudio Design & Analysis software is compatible with ThermoFisher Scientific’s line of QuantStudio Real-Time PCR systems for target sample standard curve, comparative Ct, genotyping, presence/absence and melt curve experiments. The adapter is expected to recognize the detection modality used to generate the data and generate the qPCR ASM .json file using the appropriate measurement schema.

Example I/O Files to ASM JSON

The adapter is expected to produce the following output based on the provided input:

Example1 (Multi-Analysis):

  • Input: example_qpcr_data_20240319_122427.xlsx
  • Output: example_qpcr_data_20240319_122427.json

Example2 (Genotyping):

  • Input: appbio_quantstudio_designandanalysis_QS7Pro_Genotyping_example07.xlsx
  • Output: appbio_quantstudio_designandanalysis_QS7Pro_Genotyping_example07.json

Additional Example Files:

  • Presence/Absence
  • Standard Curve
  • Relative Standard Curve
  • Relative Quantification
  • Melt Curve
  • Genotyping w/OpenArray Block

Note: Example JSON files contain only a limited amount of the instrument file data as a general guide for mapping to ASM. Examples may have missing information for calculated data document fields, however, adapter is expected to return all instrument data described in this requirements document.

Key Differences of QuantStudio Design & Analysis to QuantStudio RT-PCR

The QuantStudio Design & Analysis software export will have a few key differences compared to ThermoFisher Scientific’s similar QuantStudio RT-PCR software. These include export into .xslx format with multi-sheet results format, Cycle Threshold Results are named in the file as ‘Cq’ rather than ‘Ct’, and exports do not contain a ‘Experiment Type’ field for associating experiment-specific data.

  • Export Formats (including file extension, delimiter, software-defaults)

Handling Experiment Types and Multi-Sheet Excel File from QuantStudio Design & Analysis

In QuantStudio Design & Analysis software’s instrument file export there will not appear a field listed as ‘Experiment Type’ as previously used as a reference for associating the type of experiment and processed/calculated data document fields in QuantStudio RT-PCR software. With this field not being present, associating of measurement document, processed data document and calculated data document fields by experiment type run by a user should be identified by the presence of sheet names within the excel output file. Below are the standard sheet names that will be exported by the software based on experiment type. Experiment specific fields are described below in measurement document and processed/calculated data document tables.

  • QuantStudio Design & Analysis exports a multi-sheet Excel file .xslx which includes a base export of:
  • Amplification Data
  • Multicomponent
  • Raw Data
  • Replicate Group Result
  • Results
  • Sample Group Result
  • Additional sheets will be added based on the experimental analyses performed:
  • Standard Curve Experiment
  • Standard Curve Result
  • Relative Quantification/Relative Standard Curve Experiment
  • RQ Replicate Group Result
  • Melt Curve Experiment
  • Melt Curve Raw
  • Melt Curve Result
  • Genotyping Experiment
  • Genotyping Result
  • Presence/Absence Experiment
  • Sample Call
  • Well Call
  • Target Call
  • Control Status

Additionally, multiple analyses can be performed by a user in QuantStudio Design & Analysis for an experiment. This will result in multiples of the experiment analysis result sheets in the export file (base export sheets included).

Handling Open Array Card/Blocks

Open Array Cards differ in the recording of the location identifier to the standard A1, B1, C1 format. Each Array Card contains 48 standard wells (A1, B1, C1) with an additional 64 “sub-wells” per well, totaling 3,072 well count (denoted as A1a1, A1b1, etc.). For handling of measurement documents of experiments that use Array Cards, record each separate “sub-well” (eg. A1a1) as a separate measurement document. The location identifier should then be recorded as the Well Position column value for each row in the Results block. While each separate “sub-well” should be recorded in a separate measurement document, the expectation is that these would all be part of the measurement aggregate document for the qpcr document for the “parent” well (e.g. A1).

  • validation sample role - A validation sample role is a sample role of a material sample that is used for validation purposes.

  • experiment sample role - An experiment sample is a role that is played by a material object of interest in an assay used to obtain generalizable information about the sample source.

  • sample role - A sample role type is a classification datum that classifies samples by the type of sample role in the experiment

  • spiked sample role - A spiked sample role is a sample role of a material sample that does contain a spike substance.

  • blank role - A blank role is a sample role of a material that does not contain any analyte.

  • quality control sample role - A quality control sample role is a sample role of a material sample that is used for purposes of quality control.

  • reference sample role - A reference sample role is a sample role of a material sample that does contain a known amount of analyte.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Formatting of ‘Genotyping’ experiments

The Genotyping experiments have (2) Target DNA description per well, similar to other multiplexing experiment. The data for each Target DNA description is reported on the separate rows, similarly as is done with other multiplexing experiments, with the location identifier being the same for the two rows. To conform to the pattern used in the ASM for all other experiment types, these fields need to be segmented to separate measurement documents (within the same qPCR document).

With the Genotyping experiments, the two Target DNA description are referred to as Allele 1 and Allele 2. The Target field in the Results sheet, Results block will appear with a concatenation of target DNA name and allele1/2 (eg. C__10048053_10-Allele 1). Include entire string as Target DNA description.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the instrument software file export.

All calculated data documents should be placed within the calculated data aggregate document within the qPCR aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same qPCR document, or from measurement documents from different qPCR documents.

The calculated data document may contain a data processing document which will capture records needed to interpret how the calculation was performed. For the qPCR ASM, this data processing document could include reference DNA description and reference sample description which are needed to interpret Delta Ct and Delta Delta Ct calculations (discussed below).

Relationship between calculated values:

cycle threshold result (measurement doc) → Ct Mean → Delta Ct Mean → Delta Delta Ct → RQ

cycle threshold result (measurement doc) → Ct Mean → Equivalent Ct Mean -> Adjusted Eq Mean -> Delta EqCt Mean → Delta Delta EqCt → RQ -> RQ min/RQ Max

Sample Calculation of Delta Delta Ct and RQ

Delta Ct = Ct Mean - Ct Mean (ref target)

Delta Delta Ct = Delta Ct - Delta Ct (ref sample)

RQ = 2^ -(Delta Delta Ct)
