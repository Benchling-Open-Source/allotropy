The Thermo Absolute Q adapter is intended to convert text file (.csv) exports from QuantStudio Absolute Q Digital PCR Software to the dPCR Allotrope Simple Model (ASM). Absolute Q PCR systems are digital PCR (dPCR) instruments that are used to quantify specific DNA oligonucleotides within samples that have been ‘partitioned’ into a large number of micro-reactions.

Required User Actions

The required user actions are based on instructions provided within the QuantStudio Absolute Q Digital PCR System User Guide (MAN0025621 Rev E).

The user is expected to enter Sample Name values within the Absolute Q UI under the ‘SETUP’ tab that conform to entity identifiers within Benchling, either existing entity IDs or entities they wish to register upon processing of results. The Absolute Q software does not provide a utility for importing Sample Names from a file.

Alternatively, the user should create a plate within Benchling that uses the same barcode as the Absolute Q plate, and fill that plate with the entities being assayed.

Note: The Absolute Q plates are pre-barcoded, however they are a 16 well (4 well x4 column) plate and thus would need to be modeled as a partial 24-well plate within Benchling

If the user intends for samples to be treated as technical replicates, the sample should be given the same Sample Name and add to the same Group

The user must also specify the samples as ‘Replicates’ within the ‘Edit Groups’ modal.

The user is expected to generate a .csv export from the ‘RESULTS’ tab by clicking the download icon and selecting ‘Download CSV’.

Despite formatting the export as a .csv file, the Absolute Q software does not include the ‘.csv’ file extension. Therefore the .csv must be added when saving the exported file.

Generation of the ASM file

The Allotrope ASM for qPCR has the following structure

dPCR Aggregate Document

⮑ Device System Document

⮑ Data System Document

⮑ dPCR Document

⮑ Measurement Aggregate Document

⮑ Measurement Document(s)

⮑ Sample Document

⮑ Device Control Aggregate Document

⮑ Device Control Document(s)

⮑ Processed Data Aggregate Document

⮑ Processed Data Document

⮑ Data Processing Document

⮑ Calculated Data Aggregate Document

⮑ Calculated Data Document

⮑ Data Processing Document

The Absolute Q export .csv file has column headers which correspond to different entries within ASM.

dPCR Aggregate Document

The dPCR Aggregate Document will contain the Device System Document , Data System Document, dPCR Document, and where appropriate a Calculated Data Aggregate Document for recording processed data derived from multiple dPCR Documents.

Device System Document

The device system document within the ASM is intended to capture metadata about the instrument from which the data was obtained.

The device identifier is recorded within the Instrument column within the export file. The expectation is that this record will be the same for all rows of the file.

The brand name is “QuantStudio Absolute Q Digital PCR System”, while the product manufacturer is “ThermoFisher Scientific”.

All other values within the Device System Document can be ignored at this time.

Data System Document

The data system document within the ASM is intended to capture information about the instrument file, its location, source of generation, the adapter/converter used to generate the ASM file.

The data system instance identifier is intended to reference the computer/workstation/file-store on which the instrument file resides. This information is not included in the export, but if possible should be obtained by the adapter querying the identifier of the local system on which it is installed.

The file name is the name of the instrument file which the adapter is using as input.

The UNC path should be the complete file path of the instrument file.

The software name and software version refer to the application that generated the instrument file. This information is not included in the Absolute Q export, therefore ‘QuantStudio Absolute Q Digital PCR Software’ can be used as the default name for software name while software version is not provided and thus can be excluded from the ASM at this time.

The ASM converter name and ASM converter version refers to the the adapter being described in this requirements document used generate the ASM file from the instrument file. Therefore, the adapter should populate these fields with its own name and version.

dPCR Document

The dPCR Document is an example of a technique document in the Allotrope framework, and meant to represent each time the dPCR technique is perform. In this context the dPCR technique is performed for each plate well in the Absolute Q export - as the instrument utilizes multi-well plates. The Absolute Q file export is structured with data from each well on a single line, therefore, the information on each line in the file will recorded within a separate dPCR document - with the exception of the lines containing aggregate ‘group’ data (discussed below).

The dPCR Document contains details about: analyst - the individual who executed the procedure (when provided), and the submitter - the individual who requested the procedure and submitted samples to the analyst (when provided), a Measurement Aggregate Document, and in some cases where there is group data - a Calculated Data Aggregate Document.

The Absolute Q export does not provide records that correspond to analyst or submitter and therefore, these two fields can be ignored.

Measurement Aggregate Document

The Measurement Aggregate Document contains details about the analytical method identifier, experimental data identifier, experiment type, container type, plate well count, well volume, and details about the individual measurements within a Measurement Document array.

The experimental data identifier should be taken from the entry in the Run column of the Absolute Q export file.

The container type can be listed as “well plate” and the plate well count can be set as 16.

The Absolute Q export does not provide information corresponding to analytical method identifier, experiment type, or well volume, so these fields can be ignored.

Measurement Document

Each Measurement Document in the dPCR Document would correspond to the Target DNA being amplified in the PCR reaction. dPCR reactions, similar to qPCR reactions, can be multiplexed with amplifications of multiple DNA targets within single reaction each detected with a different reporter dye. In these cases, each DNA target : reporter dye pair would represented as a separate Measurement Document within the Measurement Aggregate Document. The Absolute Q system supports the detection of (4) reporter dyes, with a 5th dye used as the reference dye. Therefore, each dPCR Document will have either a single Measurement Document for single-plex assays or up to (4) Measurement Documents for multiplex assays.

The measurement identifier is not derived from the instrument file and instead should be a universal unique identifier generated and added to the ASM file.

The Absolute Q export only includes a date on which the run was performed, but not a time. The date is reported in a MM/DD/YYYY format. Therefore, the measurement time can be taken as the entry within the Date column and 00:00:00 can be used as the HH:MM:SS, e.g. 2023-05-23T00:00:00.

The target DNA description corresponds to the entry in the Target, while the total partition count is provided under the Total column

Sample Document

The Sample Document contains details of the sample being assess as part of the measurement.

The sample identifier, well location identifier, and well plate identifier are recorded within the Name, Well, and Plate columns.

The batch identifier and sample role type properties are not reported in the Absolute Q export.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was perform.

The device type in all cases can be recorded as “dPCR'”, while the reporter dye setting corresponds to the Dye column.

No other properties within the device control document are provided within the Absolute Q export, so these fields can be ignored.

Processed Data Aggregate Document / Processed Data Document

The processed data document contains results data which is derived from measurement data and who values are dependent on user defined parameters or transformation. The parameters which determine the results within the processed data document are recorded within the data processing document.

The entries in the Absolute Q export under the Conc. cp/uL and Positives column should be recorded in the number concentration and positive partition count fields within the processed data document.

These values are dependent upon the threshold value applied to the fluorescence signal measured by the Absolute Q instrument. While this threshold value is set for each reporter dye within the UI, it is not included in the export. If it were, then these would be recorded within a data processing document in the fluorescence intensity threshold setting.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the Absolute Q file export.

The calculated data aggregate document contains a collection of calculated data documents. The calculated data aggregate document should be placed within the technique aggregate document - in this case the dpcr aggregate document.

The calculated data document includes a calculated data identifier, which similar to the measurement identifier is not derived from the instrument file and instead should be a universal unique identifier generated and added to the ASM file.

The calculated data document also includes a data source aggregate document/data source document which details the inputs to the calculation including: a data source identifier used to reference another measurement identifier or calculated data identifier, and a data source feature which is used to reference the field within the source document which was used as input to the calculation.

The calculated data document may contain a data processing document which records information needed to interpret how the calculation was performed.

The calculated data document also includes a calculated data name, a name for the calculated result; optionally a calculated data description to record how the calculation was performed; and a calculated datum to record the calculated result.

Copy Number Variation

Digital PCR instruments, including the Absolute Q system, are commonly used to perform Copy Number Variation (CNV) analysis, which determines the number of copies of a gene within a sample. The results of CNV analysis would be reported with a calculated data document since these data are derived from comparing the concentration of DNA from one target gene to the concentration of DNA of a reference gene.

However, we do not currently have examples of Absolute Q exports from experiments in which CNV analysis was performed on which to base the data parsing.

Replicates and Pooled Results

The Absolute Q software allows for the definition of ‘groups' of wells within the 16 wells within the plate. The groups and constituent wells are designated within the Group column.

The user can specify that the wells within the group be treated as: Individual, Replicates, or Pooled. When wells are specified as Individual no aggregation of the data is performed. When wells are specified as Replicates aggregate statistics of Average Total; Average Positives; as well as Average, SD, and CV of Concentration are calculated. Finally, since dPCR relies on spatially distributed features, when wells are specified as Pooled the data is treated as though it came from one, larger well.

In all cases, the Absolute Q export includes rows with summary group data, which are interleaved amongst the rows containing data from individual wells. There ‘group rows’ are distinguished by an absence of an entry in the Name column and an entry in the Group column recording the group and number of constituent wells in parenthesis, e.g. Group 1 (8).

In the cases in which samples were designated as Individual, these ‘group rows’ contain no additional information and can be ignored.
