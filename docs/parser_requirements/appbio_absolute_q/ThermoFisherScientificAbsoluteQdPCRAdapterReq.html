Background Section

The Thermo Absolute Q adapter is intended to convert text file (.csv) exports from QuantStudio Absolute Q Digital PCR Software to the dPCR Allotrope Simple Model (ASM). Absolute Q PCR systems are digital PCR (dPCR) instruments that are used to quantify specific DNA oligonucleotides within samples that have been ‘partitioned’ into a large number of micro-reactions.

dPCR Document

The dPCR document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the dPCR technique is performed. In this framework the dPCR  technique is performed for each plate well (rather than plate) in the instrument software export. This means that information from each well should be recorded in a separate dPCR document.

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the instrument software file export.

All calculated data documents should be placed within the calculated data aggregate document within the dPCR aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same dPCR document, or from measurement documents from different dPCR documents.

Copy Number Variation

Digital PCR instruments, including the Absolute Q system, are commonly used to perform Copy Number Variation (CNV) analysis, which determines the number of copies of a gene within a sample. The results of CNV analysis would be reported with a calculated data document since these data are derived from comparing the concentration of DNA from one target gene to the concentration of DNA of a reference gene.

Replicates and Pooled Results

The Absolute Q software allows for the definition of ‘groups' of wells within the 16 wells within the plate. The groups and constituent wells are designated within the Group column.

The user can specify that the wells within the group be treated as: Individual, Replicates, or Pooled. When wells are specified as Individual no aggregation of the data is performed. When wells are specified as Replicates aggregate statistics of Average Total; Average Positives; as well as Average, SD, and CV of Concentration are calculated. Finally, since dPCR relies on spatially distributed features, when wells are specified as Pooled the data is treated as though it came from one, larger well.

In all cases, the Absolute Q export includes rows with summary group data, which are interleaved amongst the rows containing data from individual wells. There ‘group rows’ are distinguished by an absence of an entry in the Name column and an entry in the Group column recording the group and number of constituent wells in parenthesis, e.g. Group 1 (8).

In the cases in which samples were designated as Individual, these ‘group rows’ contain no additional information and can be ignored.

In cases in which samples were designated as Replicates, the Well column will contain the entry ‘Average’ and addition columns for SD and CV% will be present. In these case (5) calculated data documents should be generated per group as detailed in the table below:
