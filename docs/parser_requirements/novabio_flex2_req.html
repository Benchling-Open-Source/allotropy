NovaBio Flex Adapter Requirements

The NovaBio Flex adapter is intended to convert .csv file exports from NovaBio Flex cell culture analyzers to an Allotrope Simple Model (ASM) file for cell culture analyzers.

The adapter is expected to convert the .csv export file to an Allotrope ASM .json file

Required User Actions

The user is expect to generate a SampleResults .csv export file following the instructions detailed in section 2-32 of the NovaBio BioProfile Flex2 Quick Reference Guide (57960F 2023-03).

For the purposes of the integration, the user is expected to select either MM DD YYYY or YYYY MM DD as the date format (but not DD MM YYYY), detailed in section 3-24 of the NovaBio BioProfile Flex2 Quick Reference Guide.

The user is then expected to place the SampleResults export file in a target directory which is specified during Connection configuration, when creating a NovaBio Flex Connection within the Benchling UI.

The user can have the export file sent directly to that target directory by following the instructs for networking and shared file access in section 2-35 of the Reference Guide.

If the user wishes to capture a device identifier (discussed below), than the user should configure an Analyzer ID within the settings menu of the Flex instrument, so that this value is included within the export file name.

Generation of the ASM file

The Allotrope ASM for Cell Culture Analyzers has the following structure

⮑ Measurement Aggregate Document

⮑ Sample Document

⮑ Device Control Aggregate Document

⮑ Device Control Document(s)

⮑ Analyte Document

Device System Document

The device system document within the ASM is intended to capture metadata about the instrument from which the data was obtained.

The SampleResults export file includes a device identifier within the filename, if this value is configured by the user. If present the device identifier will appear just ahead of the timestamp.

Example: SampleResultsT261210100C2022-02-07_160455.csv

In this example. ‘T261210100C’ would be the device identifier. If present within the export filename, this parameter should be recorded within the ASM file.

The product manufacturer should be listed as “Nova Biomedical”.

All other fields within the Device System Document, including device serial number can be ignored.

Data System Document

The data system document within the ASM is intended to capture information about the instrument file, its location, source of generation, the adapter/converter used to generate the ASM file.

The data system instance identifier is intended to reference the computer/workstation/file-store on which the instrument file resides. This information is not included in the BioHT export, but if possible should be obtained by the adapter querying the identifier of the local system on which the Gateway hosting the adapter is installed.

The file name is the name of the instrument file which the adapter is using as input.

The UNC path should be the complete file path of the instrument file.

The software name and software version refer to the application that generated the instrument file. This information is not included in the SampleResults file, and thus can be ignored.

The ASM converter name and ASM converter version refers to the the adapter being described in this requirements document used generate the ASM file from the instrument file. Therefore, the adapter should populate these fields with its own name and version.

Cell Culture Analyzer Document

The Cell Culture Analyzer Document is an example of a technique document in the Allotrope parlance, and meant to represent each time a sample is assessed using a cell culture analyzer. The SampleResults file is structured with a single per sample, and therefore each line in the export will correspond to a Cell Culture Analyzer Document in the ASM.

The Cell Culture Analyzer Document contains a Measurement Aggregate Document, details about analyst - the individual who executed the procedure, and the submitter - the individual who requested the procedure and submitted samples to the analyst.

The analyst field corresponds to the system user recorded in the final (tenth) column of the archive file info row, shown as ‘ADMIN’ in the examples. As there is only a single analyst entry per export file, the analyst will be repeated for all Cell Culture Analyzer Documents in the ASM.

The analyst field corresponds the value recorded under the Operator column within the SampleResults file itself.

There is not a record of sumbitter and thus this can be ignored.

Measurement Aggregate Document

NovaBio Flex SampleResults export files follow a standard filename convention of: SampleResultsYYYY-MM-DD_HHMMSS.csv

Note: in the event that the file name does not conform to this convention, then an error message should state:

“File name is not valid. File name expect to have format of SampleResultsYYYY-MM-DD_HHMMSS.csv or SampleResults<Analyzer ID>YYYY-MM-DD_HHMMSS.csv where <Analyzer ID> is defined in Settings

The data processing time should be parsed from the timestamp provided within the SampleResults export filename.

All times recorded in the ASM file should be expressed in the timezone when timezone information is provided. Otherwise, the time should be reported as expressed within the file.

Measurement Document

Each row within the SampleResults export file refers to a sample while the columns refer to sample metadata and measured properties derived from that sample. Note, not SampleResult files will have all of the measurements which could be recorded within the ASM file. Further, not all columns within the SampleResults export file need to be recorded within the ASM file, for examples properties such as vessel pressure or sparging O2% do not need to be recorded.

The measurement identifier should be a universal unique identifier generated and added to the ASM.

The measurement time field should be obtained from the value within the Data & Time column within the SampleResults export. The NovaBio Flex allows for 3 date format options: MM DD YYY, DD MM YYYY, and YYYY MM DD; and options for time to be set in 24HR or 12HR formats. For the purposes of the integration, the user is expected to select either MM DD YYYY or YYYY MM DD as the date format (but not DD MM YYYY). In the ASM measurement time should be recording in a YYYY-MM-DD HH:MM:SS format, were time is formatted to 24-hour, inclusion of T and Z designations can be optionally included if present.

Sample Document

The sample identifier should be obtained from the Sample ID column.

The batch identifier should be obtained from the Batch ID column.

The sample role type should be obtained from the Sample Type column

Device Control Aggregate / Device Control Doc

The standard format of the ASM contains a Device Control Aggregate and Device Control Documents to capture parameters used in the data acquisiton.

No such parameters are included in the Cedex BioHT export, and those only device type of “cell culture analyzer” needs to be recorded.
