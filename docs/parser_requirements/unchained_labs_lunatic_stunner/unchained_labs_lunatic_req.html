Unchained Labs Lunatic Plate Reader Adapter Requirements

The Unchained Labs Lunatic Adapter is intended to covert .csv file exports from Unchained Labs UV-Vis plate Reader data to an Allotrope Simple Model (ASM) file for UV-VIS analysis.

Adapter Purpose:

  • Watch a designated file directory for an excel file matching the pattern of Lunatic .csv files
  • Convert the Lunatic .csv file to an Allotrope plate reader ASM .json file
  • Return the .csv file and convert the ASM .json file to a designated S3 bucket.

The adapter is expected to produce the following output based on the provided input file types:

Exported File Types:

These are the exported file types from the Lunatic Stunner Software (note: we have example data with the proposed allotrope model linked below)

  • .CSV - preferred file type

Required User Actions

The required user actions are centered around exporting a compatible file (.csv) into the expected format.

The user will open the Lunatic & Stunner Client and select their assay type (DNA/Protein/Oligo). This will determine the column that can be exported, details about the columns generated for each assay type are detailed below.

The adapter is expected to support .csv file exports from the following assay types:

  • dsDNA Turbidity
  • ssDNA Turbidity
  • RNA Turbidity
  • Protein Turbidity

The user is then expected to place this Lunatic file in a target directory which is specified during Connection configuration, when creating a Lunatic Connection within the Benchling UI.

As of now we are focusing our efforts in ensuring that we are able to adapt both DNA quantitation and Protein Quantitation via UV-Vis absorbance from our model.

Device System Document

The device system document within the ASM is intended to capture metadata about the instrument from which the data was obtained.

The fields within this document can include:

  • asset management identifier
  • description
  • brand name
  • product manufacturer
  • device identifier (required)
  • model number (required)
  • equipment serial number
  • firmware version

The Instrument ID column of the export will contain the device identifier .

The model number is always going to be Lunatic, and the product manufacturer is always going to be “Unchained Labs”.

Data System Document

The data system document within the ASM is intended to capture information about the instrument file, its location, source of generation, the adapter/converter used to generate the ASM file.

The fields within this document can include:

  • data system instance identifier
  • file name
  • UNC path
  • software name
  • software version
  • ASM converter name
  • ASM converter version

The Data System document will pull the filename into the file name field. The instrument software will always be “Lunatic Stunner Analysis”.

The file name is the name of the instrument file which the adapter is using as input.

The UNC path should be the complete file path of the instrument file. The Lunatic software can include a file path in the header portion. However, this may not coincide with the location the adapter picks-up the file, therefore the adapter should record the path of the ‘pick-up’ location.

The software name will be always “Lunatic and Stunner Analysis”.

The ASM converter name and ASM converter version refers to the adapter being described in this requirements document used to generate the ASM file from the instrument file. Therefore, the adapter should populate these fields with its own name and version.

Plate Reader Document

The plate-reader document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the plate-reader technique is performed. In this framework the plate-reader technique is performed for each plate well (rather than plate) in the Lunatic export. This means that information from each well should be recorded in a separate plate-reader document.

The fields within this document can include:

  • measurement aggregate document (required)

Measurement Aggregate Document

The measurement aggregate document contains records of the measurements performed, each captured in a measurement document as metadata about the measurements performed. Note that the measurement aggregate document and constituent documents (e.g. measurement document, sample document) within the Plate-Reader ASM inherits generic fields from the core ASM schema as well as specific fields from the plate-reader ASM schema.

The fields within this document can include:

  • measurement time (required)
  • analytical method identifier
  • experimental method identifier
  • experiment type
  • container type
  • plate well count (required)
  • well volume
  • measurement document (required - per core schema)

The measurement time is made up of the concatenation of the Date and Time columns in the file export.

If either the Date or Time not included, then the following error message should be returned:

“The file is required to include both ‘Date’ and ‘Time’ columns”

The measurement time should be recorded in a YYYY-MM-DD HH:MM:SS format, were time is formatted to 24-hour, inclusion of T and Z designations can be optionally included if present, otherwise the timezone should be expressed as provided.

The analytical method identifier should be populated from the application column.

The container Type should always be defaulted to “well plate“.

Note: Some of the example files above do not have these columns, but the customer exports will be available in future versions.

Measurement Document

The measurement document is an array of objects, each containing details about individual measurements recorded from each well within the EnVision export.

The fields within this document can include:

  • measurement identifier (required)
  • sample document (required - per core schema)
  • device control aggregate document (required - per core schema)
  • compartment temperature (required)
  • measurement fields from ultraviolet-absorbance schema - as the Lunatic is an absorbance-only instrument

Within each measurement document, there is a device control which specifies the detector wavelength and units. The detector wavelength will be represented by an integer and the units are always nm (ex. A280 nm). We pull these values from columns containing A### where ### = number (ex. A260).

If there is no column corresponding to A###, then the following error message should be returned:

“The file is required to include an absorbance measurement column (e.g. A280)”

(Note: This may change in the future if we are able to get an agreement from the end-users using the Lunatic Software)

Sample Document (within Measurement Document)

The sample document within the ASM is intended to capture metadata about the submitted samples. The location identifier and well plate identifier are populated by the Plate Position and the Plate ID columns respectively. If applicable, the sample identifier will be also pulled into the ASM file if indicated by the Sample Name column.

Device Control Aggregate Document / Device Control Document

For device type “plate reader” can be used as the default value as it is a required field in the ASM.

Remainder of Measurement Document

The value for absorbance should be taken from the entry under the A### column.

There is information provided for compartment temperature in the Lunatic export, since this is a required field it can be recorded as “null”.

Calculated Data Document

This calculated data document will capture the reported concentrations for the DNA quantitation and Protein quantitation acquisition methods, as well as ratios calculated between absorbance measurements.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the Lunatic file export.

All calculated data documents should be placed within the calculated data aggregate document within the plate reader aggregate document.

Within the supported assays the entries under the following columns should be recorded as calculated result within calculated data documents:
