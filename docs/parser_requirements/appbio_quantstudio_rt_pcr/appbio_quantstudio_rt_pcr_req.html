Thermo QuantStudio Real-Time PCR Software Adapter Requirements

Revision 05

Date: 2024-01-19

The Thermo QuantStudio adapter is intended to convert text file (.txt) exports from QuantStudio RealTime PCR Software to the qPCR Allotrope Simple Model (ASM). Thermo QuantStudio PCR systems are quantitative PCR (qPCR) - also known as real-time PCR - and are used to detect the presence and absence of specific DNA oligonucleotides. The adapter is expected to parse data from the instrument pertaining to the amplification of target amplicons and associated results fields.

The adapter is expected to:

  • Watch a designated directory for .txt files matching the pattern of QuantStudio exports
  • Convert the exported .txt file to an Allotrope .json file
  • Return the original .txt file and the converted ASM .json file to a designated S3 bucket

Required User Actions

The required user actions are based instructions provided in the QuantStudio Real-Time PCR Software Users Manual (PN 4489822, Rev A).

The user is expected to enter Sample Name values within the QuantStudio UI that conform to entity identifiers within Benchling, either existing entity IDs or entities they wish to register upon processing of results. The user may do this by manually entering Sample Names or Importing a Sample Definition File (see p.59 IFU).

The user is expect to generate a .txt export from the Export view within the QuantStudio UI.

The user can select the data to include within the export by selecting the individual tabs.

The adapter is able to process the data within the: Sample Setup, Amplification, Multicomponent, and Results tabs

The user is expected keep the default option to Export Data To One File

The user should select .txt as the file type. Processing of the .xls and .xlsx formats is not currently supported by the adapter.

The user should set the Export File Location to be the target directory specified during Connection configuration.

In order to insure that the exported files are consistently placed in the target directory for the Connection, the default export location can be set within the QuantStudio preferences menu.

The QuantStudio export file has a number of ‘blocks’ which contain relevant measurement and metadata required for the ASM. The ‘blocks’ follow the pattern of:

header - detailing device, file, and experiment information

qPCR Aggregate Document

The qPCR Aggregate Document will contain the Device System Document , Data System Document, qPCR Document, and where appropriate a Calculated Data Aggregate Document for recording processed data derived from multiple qPCR Documents.

Device System Document

The device system document within the ASM is intended to capture metadata about the instrument from which the data was obtained.

The details for device identifier, model number, and device serial number are all provided within the header block.

The device identifier is recorded within the Instrument Name field. While this field defaults to the instrument serial number, it should be recorded within the device identifier field even when the two are repeated.

Meanwhile, the model number is provided within the Instrument Type field and device serial number is provided within the Instrument Serial Number field.

All other values within the Device System Document can be ignored at this time.

Data System Document

The data system document within the ASM is intended to capture information about the instrument file, its location, source of generation, the adapter/converter used to generate the ASM file.

The data system instance identifier is intended to reference the computer/workstation/file-store on which the instrument file resides. This information is not included in the export, but if possible should be obtained by the adapter querying the identifier of the local system on which it is installed.

The file name is the name of the instrument file which the adapter is using as input.

The UNC path should be the complete file path of the instrument file.

The software name and software version refer to the application that generated the instrument file. This information is not included in the QuantStudio export, therefore ‘QuantStudio Real-Time PCR Software’ can be used as the default name for software name while software version can be excluded from the ASM at this time.

The ASM converter name and ASM converter version refers to the the adapter being described in this requirements document used generate the ASM file from the instrument file. Therefore, the adapter should populate these fields with its own name and version.

qPCR Document

The qPCR Document is an example of a technique document in the Allotrope parlance, and meant to represent each time the qPCR technique is perform. In this framework the qPCR technique is performed for each plate well in the QuantStudio export - as qPCR instruments typically rely on multi well plates. The blocks within the QuantStudio export will reference Well or Well Position (depending on the block) - these should then correspond to individual qPCR Documents.

The qPCR Document contains a Measurement Aggregate Document, details about analyst - the individual who executed the procedure, and the submitter - the individual who requested the procedure and submitted samples to the analyst.

The entry for analyst is recorded under the Experiment User Name in the header block. If no entry is provided - ‘NA’ - then this field can be ignored.

There is no entry corresponding to submitter within the QuantStudio export.

Measurement Aggregate Document

The Measurement Aggregate Document contains details about the analytical method identifier, experimental data identifier, experiment type, container type, plate well count, well volume, and details about the individual measurements within a Measurement Document array.

The QuantStudio export does not provide a field corresponding to analytical method identifier, so this entry can be ignored.

The experimental data identifier should be taken from the Experiment Name within the header block.

The experiment type is recorded within the Experiment Type field within the header block. The record of Experiment Type should correspond to one of the following:

  • Standard Curve
  • Relative Standard Curve
  • Comparative Ct (ΔΔCt)
  • Melt Curve
  • Genotyping
  • Presence/Absence

The container type in all cases should be “qPCR reaction block”, while the plate well count should be taken from the Block Type field in the header block. The value of plate well count should be either 96 or 384.

The well volume is not reported in the QuantStudio export, and can be ignored.

Measurement Document

Each Measurement Document in the would correspond to the Target DNA being amplified in the PCR reaction. qPCR reactions can be multiplexed with amplifications of multiple DNA targets within single reaction, with each represented as a separate Measurement Document within the Measurement Aggregate Document. In practice this multiplexing is typically limited to (2) DNA targets. Therefore, each qPCR Document will have either a single Measurement Document for single-plex assays or two Measurement Documents for multiplex assays.

The measurement identifier is not derived from the instrument file and instead should be a universal unique identifier generated and added to the ASM file.

The measurement time is recoded within the Experiment Run End Time field in the header block and should be recorded in a YYYY-MM-DD HH:MM:SS format.

The target DNA description corresponds to the entry in the Target Name column within the Sample Setup block. In the case of a multiplexed experiment there will be multiple entries per Well in the Sample Setup block each with different DNA targets.

Sample Document

The Sample Document contains details of the sample being assess as part of the measurement.

The sample identifier, sample role identifier, and well location identifier, are recorded within the Sample Setup block, with the corresponding entries included under the Sample Name, Task, and Well Position columns.

The well plate identifier if present will be recorded in the Experiment Barcode field in the header block.

The batch identifier and mass concentration properties are not reported in the QuantStudio export.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was perform.

The device type in all cases can be recorded as “qPCR”.

The device identifier has been captured previously in the Device System Document and does not need to be recorded again here. The option for device identifier is provided at this level of the data model if one desires to capture the identify of a specific instrument component, that is a part of the larger device system, which is used to perform the measurement.

The measurement method identifier is obtained from the Quantification Cycle Method within the header block.

The total cycle number setting is not reported directly, but this value can be determined by taking the largest number recorded within the Cycle column within the Amplification Data block. Note, that the Amplification Data block is indexed using Well rather than Well Position, where Well values of 1, 2, 3 correspond to Well Position values of A1, A2, A3.

Besides total cycle number setting, the other PCR thermo-cycling parameters are not included in the QuantStudio output, so the entries for denaturing, annealing, and extension can be ignored.

The record for qPCR Detection Chemistry is provided in the Chemistry field within the header block.

The reporter dye setting and quencher dye setting are provided within the Reporter and Quencher columns within the Sample Setup block OR the Results block.

The passive reference dye setting is provided in the Passive Reference field within the header block.

Processed Data Aggregate Document / Processed Data Document

The processed data document contains results data which is derived from measurement data and who values are dependent on user defined parameters or transformation. The parameters which determine the results within the processed data document are recorded within the data processing document.

Data Processing Document

The entries for the data processing document are all found within the Results block table.

The automatic cycle threshold enable setting, cycle threshold value setting, and automatic baseline determination enabled setting are provided under the Automatic Ct Threshold, Ct Threshold, and Automatic Baseline columns.

The baseline determination start cycle setting and baseline determination end cycle can be excluded from the ASM. While there are Baseline Start and Baseline End columns within the Results block table, these do not reflect the setting values for these parameters in the originating QuantStudio experiment.

The genotyping determination method, and genotyping determination method setting are only relevant in the context of Presence/Absence or Genotyping experiment types. Within Presence/Absence experiments the value within the Threshold column should be returned to the genotyping determination method setting, while for Genotyping experiments, the entry under the Method column should be returned to the genotyping determination method field.

Remainder of Processed Data Document

The cycle threshold result, normalized reported result, baseline corrected reported result, and genotyping determination result are all reported as part of the Results block.

The cycle threshold result, is recorded under the CT column and is expected to be present in any data set in which the measurement method identifier / Quantification Cycle Method is ‘Ct’.

The normalized reporter result is recorded under the Rn column while the baseline corrected reported result is recorded under the Delta Rn column. It is expected that one of these values be reported in Presence/Absence experiments.

The genotyping determination result is recorded under the Call column, and is expected to be reported in Presence/Absence and Genotyping experiments.

The data for the normalized reporter data cube and baseline corrected data cube are recorded within the Amplification Data block, with the normalized reporter data under the Rn column and the baseline corrected data under the Delta Rn column.

Remainder of Measurement Document

The remainder of the Measurement Document contains the reporter dye data cube, passive reference dye data cube, and melt curve data cube.

The data for reporter dye data cube and passive reference dye data cube are recorded within the Multicomponent block, with the column headers corresponding to the reporter dye setting and the passive reference dye setting respectively.

The data for the melt curve data cube are recorded within the Melt Curve Raw Data block. Note, that similar to the Amplification Data block, the Melt Curve raw Data block is indexed using Well rather than Well Position, where Well values of 1, 2, 3 correspond to Well Position values of A1, A2, A3.

Formatting of ‘Genotyping’ experiments

The Experiment Type Genotyping is variant of the multiplexing assays performed by QuantStudio. The file exports from Genotyping experiments have an alternative formatting of the Results block table compared other Experiment Types. The Genotyping experiments have (2) Target DNA per well, similar to other multiplexing experiments, however, the data for each Target DNA is reported on the same row, rather than on separate rows as is done with other multiplexing experiments. To conform to the pattern used in the ASM for all other experiment types, these fields need to be segmented to separate measurement documents (within the same qPCR document).

With the Genotyping experiments, the two Target DNA are referred to as Allele 1 and Allele 2. The table below details the columns within the Results block table which have the Target DNA appended.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the QuantStudio file export.

The calculated data document includes a calculated data identifier, which similar to the measurement identifier is not derived from the instrument file and instead should be a universal unique identifier generated and added to the ASM file.

The calculated data document also includes a data source aggregate document/data source document which details the inputs to the calculation including: a data source identifier used to reference another measurement identifier or calculated data identifier, and a data source feature which is used to reference the field within the source document which was used as input to the calculation.

The calculated data document may contain a data processing document which will records needed to interpret how the calculation was performed. For the qPCR ASM, this data processing document could include reference DNA description and reference sample description which are needed to interpret Delta Ct and Delta Delta Ct calculations (discussed below).

The calculated data document also includes a calculated data name, a name for the calculated result; optionally a calculated data description to record how the calculation was performed; and a calculated datum to record the calculated result.

As shown in Thermo_QuantStudio_example01_asm_rev03.json the calculated data document for Delta Delta Ct is derived from the following:

  1. A calculated data document for Ct Mean which references as data sources the cycle threshold results from all measurement documents sharing the same sample identifier and target DNA description.

Note: In this example file there are no replicates of sample identifier and target DNA, so the there is only a single data source document and the value of Ct Mean equals the cycle threshold result since this reflects a Mean of n=1

  1. A calculated data document for Delta Ct Mean which references as data sources the two calculated data documents for Ct Mean from (#1) which reference measurement documents with the same sample identifier
  2. A calculated data document for Delta Delta Ct which references as data sources the calculated data document for Delta Ct Mean from (#2) and the calculated data document for Delta Ct Mean - also from #2 - but specifically for the sample identifier that corresponds to the reference sample description

The records for reference DNA description and reference sample description appear at the bottom of the Results block. The reference DNA description appears in the field Endogenous Control and reference sample description in the field Reference Sample.

For the calculated fields: Delta Ct Mean, Delta Ct SE, Delta Delta Ct, RQ, RQ min, RQ max there will be no value reported for those rows where the Target DNA equals the Endogenous Control / reference DNA description .

The calculated data documents for other calculated fields including Y-Intercept, R2, Slope, and Efficiency will be reference as data sources the measurement documents that share the same target DNA and have the well role type of STANDARD.
