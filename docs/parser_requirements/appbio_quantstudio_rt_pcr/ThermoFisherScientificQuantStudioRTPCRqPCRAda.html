Background Section

The Thermo QuantStudio adapter is intended to convert text (.txt)/excel file (.xlsx) exports from QuantStudio RealTime PCR Software to the qPCR Allotrope Simple Model (ASM). Thermo QuantStudio PCR systems are quantitative PCR (qPCR) - also known as real-time PCR - and are used to detect the presence and absence of specific DNA oligonucleotides. The adapter is expected to parse data from the instrument pertaining to the amplification of target amplicons and associated results fields.

The adapter is expected to:

  • Watch a designated directory for .txt/.xlsx files matching the pattern of QuantStudio exports
  • Convert the exported .txt/.xlsx file to an Allotrope .json file
  • Return the original .txt/.xlsx file and the converted ASM .json file to a designated S3 bucket

⮑ Measurement Aggregate Document 1 per sample/well

⮑ Sample Document 1 per sample/well

⮑ Device Control Aggregate Document 1 per sample/well

This means that information from each well should be recorded in a separate qPCR document.

Formatting of ‘Genotyping’ experiments

The Experiment Type Genotyping is variant of the multiplexing assays performed by QuantStudio. The file exports from Genotyping experiments have an alternative formatting of the Results block table compared other Experiment Types. The Genotyping experiments have (2) Target DNA per well, similar to other multiplexing experiments, however, the data for each Target DNA is reported on the same row, rather than on separate rows as is done with other multiplexing experiments. To conform to the pattern used in the ASM for all other experiment types, these fields need to be segmented to separate measurement documents (within the same qPCR document).

With the Genotyping experiments, the two Target DNA are referred to as Allele 1 and Allele 2. The table below details the columns within the Results block table which have the Target DNA appended.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the instrument software file export.

All calculated data documents should be placed within the calculated data aggregate document within the qPCR aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same qPCR document, or from measurement documents from different qPCR documents.

The calculated data document may contain a data processing document which will records needed to interpret how the calculation was performed. For the qPCR ASM, this data processing document could include reference DNA description and reference sample description which are needed to interpret Delta Ct and Delta Delta Ct calculations (discussed below).

As shown in Thermo_QuantStudio_example01_asm_rev03.json the calculated data document for Delta Delta Ct is derived from the following:

  1. A calculated data document for Ct Mean which references as data sources the cycle threshold results from all measurement documents sharing the same sample identifier and target DNA description.

Note: In this example file there are no replicates of sample identifier and target DNA, so the there is only a single data source document and the value of Ct Mean equals the cycle threshold result since this reflects a Mean of n=1

  1. A calculated data document for Delta Ct Mean which references as data sources the two calculated data documents for Ct Mean from (#1) which reference measurement documents with the same sample identifier
  2. A calculated data document for Delta Delta Ct which references as data sources the calculated data document for Delta Ct Mean from (#2) and the calculated data document for Delta Ct Mean - also from #2 - but specifically for the sample identifier that corresponds to the reference sample description

The records for reference DNA description and reference sample description appear at the bottom of the Results block. The reference DNA description appears in the field Endogenous Control and reference sample description in the field Reference Sample.

For the calculated fields: Delta Ct Mean, Delta Ct SE, Delta Delta Ct, RQ, RQ min, RQ max there will be no value reported for those rows where the Target DNA equals the Endogenous Control / reference DNA description .

The calculated data documents for other calculated fields including Y-Intercept, R2, Slope, and Efficiency will be reference as data sources the measurement documents that share the same target DNA and have the well role type of STANDARD.

Note that the calculation of RQ differs between Comparative Ct and Relative Standard Curve experiments.

cycle threshold result (measurement doc) → Quantity → Quantity Mean → RQ
