NovaBio Flex Solution Analyzer Adapter Requirements

Revision 03

Date: 2025-03-13

Background Section

The NovaBio Flex adapter is intended to convert .csv exports from instrument software and generate a .json file corresponding to the Solution Analyzer Allotrope Simple Model (ASM). The NovaBio Flex is a cell culture analyzer instrument that supports dissolved gas, pH, cell viability, osmolality and metabolite detection measurements of bioprocess samples. Measurement results will fall under five categories of measurement schemas from the Solution Analyzer ASM (blood-gas detection, cell-counting detection, metabolite-detection, osmolality detection, pH-detection). The adapter is expected to recognize the detection modality used to generate the data and generate the solution-analyzer ASM .json file using the appropriate measurement schema(s).

Note that the NovaBio Flex often performs measurements of multiple detection modes on a single sample, thus multiple measurement schemas can be referenced in the final JSON output. This is discussed further in the documentation below.

Required User Actions

The user is expected to generate a SampleResults .csv export file following the instructions detailed in section 2-32 of the NovaBio BioProfile Flex2 Quick Reference Guide (57960F 2023-03).

For the purposes of the integration, the user is expected to select either MM DD YYYY or YYYY MM DD as the date format (but not DD MM YYYY), detailed in section 3-24 of the NovaBio BioProfile Flex2 Quick Reference Guide.

The user is then expected to place the SampleResults export file in a target directory which is specified during Connection configuration, when creating a NovaBio Flex Connection within the Benchling UI.

The user can have the export file sent directly to that target directory by following the instructions for networking and shared file access in section 2-35 of the Reference Guide.

If the user wishes to capture a device identifier (discussed below), then the user should configure an Analyzer ID within the settings menu of the Flex instrument, so that this value is included within the export file name.

Solution-Analyzer Document

The solution analyzer document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the solution-analyzer technique is performed. In this framework the solution-analyzer technique is performed for each sample in the instrument software export. This means that information from each sample should be recorded in a separate solution-analyzer document.

Measurement Aggregate Document

The measurement document is an array of objects, each containing details about individual measurements recorded from each sample within the instrument software export. Note that the measurement aggregate document and constituent documents (e.g. measurement document, sample document) within the Solution-Analyzer ASM inherits generic fields from the core ASM schema as well as specific fields from the solution-analyzer ASM schema. As mentioned above, there are additional extensions to the solution-analyzer ASM schema defined in respective measurement schemas (e.g. measurement document, device control document, analyte document, processed data document) that solution-analyzers support,  including cell-counting, metabolite-detection, osmolality, uv-absorbance, blood-gas-detection, pH detection.

Handling of Measurement Documents in NovaBio Flex

The NovaBio Flex as mentioned above can handle multiple measurement modalities to record measurement results. For recording of measurement documents within the Solution Analyzer ASM, measurement documents should be instantiated for each measurement modality (eg. cell-counting, metabolite-detection, osmolality, uv-absorbance, blood-gas-detection, pH detection) that is recorded for a sample.

For example, 4 measurement documents should be instantiated if a blood-gas detection measurement, cell-counting detection, metabolite-detection, and pH-detection measurement are recorded from the instrument file for a sample with the same sample identifier.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control documents. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Analyte Aggregate/Analyte Document (Metabolite Detection)

The analyte document within the Metabolite-detection measurement schema ASM is intended to record information about each analyte measured within the sample. As several analytes are typically measured in these assays, these are structured within an analyte aggregate document which in turn contains the analyte document. The analyte document is an array of objects, each recording details about a specific analyte.

Handling Analyte Documents for NovaBio Flex

Multiple analytes can be detected for a given sample (ie. sample identifier). For handling of multiple analyte results performed on one sample, the NovaBio Flex adapter should instantiate one analyte aggregate document and record each of the analytes identified in the instrument file for the sample as separate analyte documents.
