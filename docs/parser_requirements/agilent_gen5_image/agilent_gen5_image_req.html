Gen5 Image Adapter Requirements

Revision 06

Date: 2024-05-21

Background Section

The Agilent Gen5 Image adapter is intended to convert text file exports from the Gen5 Image Plus and Gen5 Image Prime software and generate a .json file corresponding to the plate-reader Allotrope Simple Model (ASM).

The Gen5 software supports the complete product line of BioTek brand plate readers manufactured by Agilent, which includes single-mode and multi-mode plate readers. These instruments are able to perform measurements using one (single) or more (multi) detection modality including: absorbance, fluorescence, luminescence, optical-imaging measurements. While the Gen5 Image Plus and Gen5 Image Prime packages specifically with those optical-imaging plate readers and automated microscopes.

The plate-reader ASM supports the recording of data from these different measurement modalities by referencing separate measurement schema for uv-absorbance, fluorescence, luminescence, or optical-imaging.

This adapter is distinct from the previously developed Gen5 (absorbance, fluorescence, luminescence) adapter in that it is specified for handling only the optical-imaging modality of Gen5 Image Plus and Gen5 Image Prime. The adapter is expected to recognize the optical-imaging detection modality used to generate the data and generate the plate-reader ASM .json file using the optical-imaging measurement schema.

Gen5 Image Input File Blocks

For defining sections in the example files for referencing specific fields to map to ASM parameters, it can be helpful to include a list of the blocks expected

The default Gen5 export file is a .TXT file with a number of ‘blocks’ which contain relevant measurement and metadata required for the ASM. These include a “Header”, Layout (not always present), and Results.

Gen5 Image outputs will have the same general format as the other measurement modalities (absorbance, fluorescence, luminescence). To differentiate the optical-imaging measurement schema, the adapter should look in the Procedure Details, Read field for a string value of ‘Image Single Image’, ‘Image Montage’, or ‘Image Z-Stack’. The optical-imaging measurement schema should be used to record optical-imaging specific device control settings and image feature results.

The ‘blocks’ follow the pattern of:

Header:

– Contains a number of keyword:value pairs recording metadata about the instrument, software, and measurement performed

Layout:

– Contains details related to the samples being measured. The Layout block is formatted as a plate matrix (row x column) but with every position within the matrix functioning as a list of entries - with the label for each entry in the list appearing in the right margin.

Note: Gen5 Image files will not always contain a Layout block. The adapter should be able to handle the rest of the results data/metadata within the instrument file when not present. Some of the sample document fields will be recorded in a different manner when the Layout block is not present (see sample document field mapping below).

Results:

– Contains both measurement values and calculated results corresponding to an individual well within the plate. The Results block is formatted similarly to the Layout block as a plate matrix (row x column) again with every position in the matrix functioning as a list of entries - with the label for each entry in the list appearing in the right margin

 

Note: The Gen5 Image adapter is only expected to support the Default export format, which follows this Header/Layout/Results pattern. The Default export format can be discerned by the keywords: Software Version, Experiment File Path, Protocol File Path, Plate Number, and Procedure Details allows appearing in that order from the beginning of the file on lines 3, 7, 8, 12, and 19 respectively. 

These ‘blocks’ are repeated for each plate included in the file export, however the Gen5 Image adapter is only expected to support a single plate per export file. Any file that includes multiple plates should generate an error and the associated error message (see below).

Plate-Reader Document

The plate-reader document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the plate-reader technique is performed. In this framework the plate-reader technique is performed for each plate well (rather than plate) in the instrument software export. This means that information from each well should be recorded in a separate plate-reader document.

Measurement Document

The measurement document is an array of objects, each containing details about individual measurements recorded from each well within the instrument software export.

Within Gen5 Image Prime, a user can define up to three different measurement modes (defined as Read in the instrument file): Image Single Image, Image Montage, Image Z-stacking. For each of these modes, users can also define the instrument settings* which contain metadata about the device control parameters.

Any file that does not reference “Image Single Image”, “Image Montage”, or “Image Z-stacking” should generate an error and the associated error message (see below).

*Note: Instrument settings will appear in the instrument file under Channel (Fluorescent Imaging) and/or Color Camera (Transmitted Light Imaging)

To handle this pattern for measurement modes and instrument settings, the adapter should record a new measurement document for each respective Channel and/or Color Camera string that appears in the instrument file. The associated sample document for each measurement document under a given well will remain the same, however the device control document will be expected to differ.

In the case of the above example instrument file, there should be two measurement documents recorded, with the associated metadata recorded from each.

Handling of Image Files from Gen5

The Gen5 .txt export does not provide file information about the images acquired. These files are included in a separate image file directory. Because there is no information within the instrument file itself to associate the image files for a specific well, the image document can be left unrecorded.

Handling Image Features in Gen5

Processed image feature data will be formatted the same as the Results block with the label corresponding to image feature name appearing in the right margin of the block, and the image feature result each well appearing in the corresponding position in the plate matrix.

Since the plate reader document corresponds to a well of the plate, and the image feature aggregate document / image feature documents are child objects of the plate reader document - only the image feature results for that particular well should appear within the image feature aggregate document / image feature documents under that plate reader document.

In cases where there are multiple measurement documents within a plate reader document, the image feature aggregate / image feature document should be a child of the measurement aggregate document. This is because the image features cannot be discerned to what measurement document they derive from.

In the case of a plate reader document containing only one measurement document, the image feature aggregate document / image feature document should be a child of the measurement document. The image features are known to derive from the measurement document fields.

Where possible each image feature document should have a correspondent data source document where the data source identifier should be recorded as the image file name from which the image features derives. Multiple images can be used to generate image feature(s) and should be recorded in the data source aggregate document as separate data source documents. However, since the Gen5 export does not provide information about the source image files, the data source documents for image features are not required at this time.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the instrument software file export.

All calculated data documents should be placed within the calculated data aggregate document within the plate reader aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same plate reader document, or from measurement documents from different plate reader documents.

The formatting of the Gen5 Image export does not make it possible to distinguish between those values which represent images features and those that are calculated results. Because of this, all values present in the Results block of Gen5 for imaging data are presumed to be ‘image features’ and are to be recorded as image features in the image feature document within the respective plate reader document. The corresponding consequence of this is that since no values are designated as ‘calculated data’, the calculated data aggregate document/calculated data document are not required at this time.
