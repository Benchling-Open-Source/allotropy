Thermo Fisher Scientific NanoDrop One Spectrophotometer Adapter Requirements

Revision 01

Date: 2024-08-05

Background Section

  • Thermo Fisher NanoDrop One adapter is intended to convert CSV/XLSX exports from instrument software and generate a .json file corresponding to the spectrophotometry Allotrope Simple Model
  • Thermo Fisher’s NanoDrop One Spectrophotometer instrument uses in-built NanoDrop One software. The spectrophotometer has the following operations: nucleic acid measurement, proteins measurement, measurement @ OD600, custom measurement and time-based kinetic measurements.
  • The adapter is expected to parse data from the instrument and structure the data from each sample to the associated result fields.

Example I/O Files to ASM JSON

The adapter is expected to produce the following output based on the provided input:

  • Input: thermo_fisher_nanodrop_one_example_1.xlsx
  • Output: thermo_fisher_nanodrop_one_example_1.json

Spectrophotometry Aggregate Document

The spectrophotometry aggregate document represents the highest level of organization in the spectrophotometry ASM, and will contain the device system document - for recording details about the instrument, the data system document - for recording details about the data file, the spectrophotometry document - for recording data from each well of the plate, and where appropriate a calculated data aggregate document for recording calculated results derived from multiple spectrophotometry documents (e.g. aggregated results from multiple wells).

In this framework the spectrophotometry technique is performed for each sample and/or plate well (rather than plate) in the instrument software export. This means that information from each well should be recorded in a separate spectrophotometry document.sheet_name.split(maxsplit=1)

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the instrument software file export.

All calculated data documents should be placed within the calculated data aggregate document within the spectrophotometry aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same spectrophotometry document, or from measurement documents from different spectrophotometry documents.

Key Considerations:

  1. File formats supported: .csv or .xls. The adapter will read the data from the CSV or XLS files.

  1. The differences between the NanoDrop One and NanoDrop OneC UV-Vis spectrophotometer include the NanoDrop OneC's support for cuvette sample positioning, which enables additional applications such as time-based kinetics measurement, cell culture measurements, colorimetric assays, and analysis of dilute samples, in addition to nucleic acid and protein measurements.

  1. Currently, this adapter supports parsing for nucleic acid measurements only.

  1. To ensure the adapter functions correctly, the instrument export file must contain following field names in the instrument export file for the nucleic acid measurement method. The fields are as follows:
  1. Date
  2. Sample Name
  3. Nucleic Acid(ng/uL) [The unit can vary]
  4. A260/A280
  5. A260/A230
  6. A260
  7. A280
  8. Nucleic Acid Factor
  9. Baseline Correction (nm)
  10. Baseline Absorbance

  1. To ensure the adapter functions correctly, the instrument export file must contain values for the mandatory keys defined in ASM. The required fields are as follows:
  1. measurement time - Date
  2. absorbance - Headers with A260 or A230
  3. sample identifier - Sample Name

  1. In this version, we have added few custom fields which might require extension to the ASM in the future:
  1. baseline absorbance
  2. nucleic acid factor

  1. The unit for 'nucleic acid factor' is not yet defined in the ASM (ng-cm/µL); therefore, it is currently represented as unitless.
