Vi-Cell BLU Cell Counter Adapter Product Requirements

Revision: 03

Date: 2023-07-23

The Vi-Cell BLU Cell Counter adapter is intended to convert text file exports from Beckman Vi-Cell cell counters to an Allotrope Simple Model (ASM) file for cell counters.

The adapter is expected to:

  • Watch a designated file directory for .csv files matching the pattern of Vi-Cell BLU export files
  • Convert the archive .csv file to an Allotrope ASM .json file
  • Return the original archive .csv file and converted ASM .json file to a designated S3 bucket.

The adapter is expected to produce the following output based on the provided input:

  • Input: Beckman_Vi-Cell-BLU_example01_instrumentOutput.csv
  • Output: Beckman_Vi-Cell-BLU_example01_asm_rev03.json

Additional example:

  • Input: Beckman_Vi-Cell-BLU_example02_instrumentOutput.csv
  • Output: Beckman_Vi-Cell-BLU_example02_asm_rev03.json

Required User Actions

The required user actions are based instructions provided in the Vi-Cell BLU Instructions for Use (PN C13232AH, March 2023).

The user is expected to enter Sample ID values within the Vi-Cell BLU UI that conform to entity identifiers within Benchling, either existing entity IDs or entities they wish to register upon processing of results.

The user is expect to generate a .csv export from a ‘Sample Set’ as describe in the Vi-Cell BLU Instructions for Use (IFU) or Vi-Cell BLU UI as shown below:

The user is then expected to place this .csv export file in a target directory which is specified during Connection configuration, when creating a Vi-Cell BLU Connection within the Benchling UI.

The user can specify this target directory for an individual Sample Set, by setting the directory path under the ‘Advanced Options’ (gear icon) on the Sample Set.

It is NOT RECOMMENDED that data to be ingested via the adapter be exported via the ‘append results and export to’ within Run Options under the Settings menu as this concatenates all data generated overtime and thus could result in the creation of repeated measures when processed via the Lab Auto Run.

Generation of the ASM file

The Allotrope ASM for Cell Counters has the following structure

Cell Counting Aggregate Document

⮑ Device System Document

⮑ Data System Document

⮑ Cell Counting Document

⮑ Measurement Aggregate Document

⮑ Measurement Document(s)

⮑ Sample Document

⮑ Device Control Aggregate Document

⮑ Device Control Document(s)

⮑ Processed Data Document

⮑ Data Processing Document

Cell Counting Aggregate Document

The Cell Counting Aggregate Document will contain the Device System Document , Data System Document and the Cell Counting Document.

Device System Document

The device system document within the ASM is intended to capture metadata about the instrument from which the data was obtained.

The Vi-Cell BLU .csv export from the Sample Set does not include specific device information.

The model number should be recorded as ‘Vi-Cell BLU’, while the device identifier and device serial number parameters can be excluded from the ASM file.

Data System Document

The data system document within the ASM is intended to capture information about the instrument file, its location, source of generation, the adapter/converter used to generate the ASM file.

The data system instance identifier is intended to reference the computer/workstation/file-store on which the instrument file resides. This information is not included in the Vi-Cell BLU export, but should be obtained by the adapter querying the identifier of the local system on which it is installed.

The file name is the name of the instrument file which the adapter is using as input.

The UNC path should be the complete file path of the instrument file.

The software name and software version refer to the application that generated the instrument file. This information is not included in the Vi-Cell BLU export, therefore ‘Vi-Cell BLU’ can be used as the default name for software name while software version can be excluded from the ASM at this time.

The ASM converter name and ASM converter version refers to the the adapter being described in this requirements document used generate the ASM file from the instrument file. Therefore, the adapter should populate these fields with its own name and version.

Cell Counting Document

The Cell Counting Document is an example of a technique document in the Allotrope parlance, and meant to represent each time the cell counting technique is perform. In this framework the cell counting technique is performed for each sample; the Vi-Cell BLU .csv export is structured with a single sample per line, therefore, each line in the Vi-Cell BLU export (excluding column headers) will correspond to a Cell Counting Document in the ASM.

The Cell Counting Document contains a Measurement Aggregate Document, details about analyst - the individual who executed the procedure, and the submitter - the individual who requested the procedure and submitted samples to the analyst.

The analyst field corresponds to the entry under ‘Analysis by’ column header. While this entry is repeated for all lines in the export .csv this should be a unique value per file. The default value for this field in the example files is ‘Vi-CELL’

Measurement Aggregate Document

Measurement Document

As the Vi-Cell BLU only performs one type of measurement, brightfield imaging, there will only be a single Measurement Document per Cell Counting Document. Other models of cell counters capable of performing brightfield and fluorescence measurements could have multiple Measurement Documents.

The measurement identifier is not derived from the instrument file and instead should be a universal unique identifier generated and added to the ASM file.

Measurement time should be obtained from the ‘Sample Date’ field within the Vi-Cell BLU output. In the ASM measurement time should be recording in a YYYY-MM-DD HH:MM:SS format, were time is formatted to 24-hour, inclusion of T and Z designations can be optionally included if present, otherwise the timezone should be expressed as provided.

Sample Document

The entry defined under Sample ID should be recorded in the sample identifier field in the sample document.

The Vi-Cell BLU sample results export does not include information pertaining to batch identifier, sample role type, or local identifier so these parameters can be excluded from the ASM file.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was perform. Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

As a class of instruments, cell counters will modeled as the core device - cell counter - used to perform the sample handling, and the detector(s) - brightfield and/or fluorescence imagers used to capture images of cells for processing.

As the Vi-Cell BLU is a cell counter, the ASM should include a device control document with the device type of ‘cell counter’. The sample volumne setting is the only control parameter to be include within the ‘cell counter’ device control document however this is not included in the Vi-Cell BLU export and thus can be excluded.

The Vi-Cell BLU instrument relies on brightfield imaging for measuring cells within the samples being processed. Therefore, a second device control document with the device type parameter should always be populated with ‘brightfield imager (cell counter)’ and the detection type parameter with ‘brightfield’.

Similarly, while illumination, exposure and gain are relevant to brightfield imaging, details about these settings are not included, so illumination setting, exposure duration setting, and detector gain setting can also be excluded from the ASM file.

For other models of cell counters utilizing fluorescence imaging, these would include an additional device control document with the device type parameter should always be populated with ‘fluorescence imager (cell counter)’ and the detection type parameter with ‘fluorescence’, illumination, exposure, and detector gain settings; plus additional parameters of excitation wavelength setting, emission wavelength setting , and fluorescence tag. However, since the Vi-Cell BLU lacks fluorescence detection, parameters can be excluded from the ASM file.

Processed Data Document

The processed data document contains results data generated by the cell counter. These results are classified as processed data since the values are dependent on which objects are classified as ‘cells’ by the instrument. The parameters which determine the results within the processed data document are recorded within the data processing document.

Data Processing Document

The following column headers within the Vi-Cell BLU .csv export map to the corresponding field within the data processing document. However, the user must configure the export file to include these parameters. If these are not present, then these fields can be excluded from the ASM file.

In the ASM measurement time should be recording in a YYYY-MM-DD HH:MM:SS format, were time is formatted to 24-hour, inclusion of T and Z designations can be optionally included if present, otherwise the timezone should be expressed as provided.

All density parameters are assumed to have units of 10^6 cell/mL

Conversely, the following fields are not provided within the Vi-Cell BLU .csv export:

  • dead cell density (cell counter)
  • average dead cell diameter (cell counter)
  • total cell diameter distribution data cube
  • dead cell count
  • image file

Alternative Measurement Document

Additionally, as the Vi-Cell BLU does not support fluorescence detection, the following fields represented in the alternative measurement document are not present, including:

  • fluorescence tag - captured within device control document
  • fluorescence tag positive cell count - captured within processed data document
  • fluorescence tag positive cell density - captured within processed data document
  • fluorescence tag positive cell percentage - captured within processed data document
