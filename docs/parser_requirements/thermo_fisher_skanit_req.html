ThermoFisher SkanIt Plate Reader Adapter Requirements

Revision 1

Date: May 2024

Background Section

  • The adapter is intended to convert .XLSX exports from VarioSkan instrument using SkanIt software and generate a .json file corresponding to the plate-reader Allotrope Simple Model
  • VarioSkan is a plate reader instrument from ThermoFisher that can perform fluorescence (fluorescence intensity or time-resolved fluorescence), absorbance, luminescence, and AlphaScreen (bead-based biomolecular interaction assay) measurements. The instrument can also perform environment control elements, such as change the temp, hold the sample for a time period, change the O2/CO2, shake the samples, and dispense reagents into the wells.
  • SkanIt is the software that controls the VarioSkan instrument. It allows for building protocols (and templates) and analysis (and templates). Ultimately, this software can create the report that can be exported for ingestion by Benchling.
  • The adapter is expected to recognize the detection modality used to generate the data and generate the plate-reader ASM .json file using the appropriate measurement schema.
  • More complicated analyses, such as curve fitting and custom formulas are not shown.

Plate-Reader Document

The plate-reader document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the plate-reader technique is performed. In this framework the plate-reader technique is performed for each plate well (rather than plate) in the VarioSkan SkanIt export. This means that information from each well should be recorded in a separate plate-reader document.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Processed Data Aggregate/Processed Data Document

The processed data document contains results data which is derived from measurement data with values that are dependent on user defined parameters or transformation. The parameters which determine the results within the processed data document are recorded in the data source document.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the VarioSkan SkanIt file export.

All calculated data documents should be placed within the calculated data aggregate document within the plate reader aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same plate reader document, or from measurement documents from different plate reader documents.[be][bf]
