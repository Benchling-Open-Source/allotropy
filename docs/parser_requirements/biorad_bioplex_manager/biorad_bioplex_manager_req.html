Bio-Rad Bio-Plex Manager Multi-Analyte-Profiling Adapter Requirements

Revision 01

Date: 2024-03-11

Purpose

The Bio-Rad Bio-Plex Manager multi-analyte-profiling adapter is intended to convert XML file exports from Bio-Rad Bio-Plex Manager software and generate a .json file corresponding to the multi-analyte-profiling (MAP) Allotrope Simple Model (ASM). The Bio-Plex Manager software is used in conjunction with the Bio-Rad xMAP instruments which are used to perform multiplexed, bead-based, immunoassays for detecting a variety of antigens within a sample. The adapter is expected to parse data from the instrument and structure the data from each sample to the associated result fields.

Example I/O Files to ASM JSON

The adapter is expected to produce the following output based on the provided input:

  • Input: bio-rad_bio-plex_manager_example_01.xml
  • Output: bio-rad_bio-plex_manager_example_01.json

Note the json example only includes data for the first well.

Required User Actions

The required user actions are based on instructions provided in the Bio-Plex 6.2 Manager User Guide.

Following the Document Export Options as detailed on p.198, the user should specify the Content of the export as the “Bio-Plex XML”, and specify the directory to which this file should be saved.

The user is recommended to have the XML file generated automatically at the conclusion of the instrument run by selecting this option within the Advanced Run Settings.

Multi Analyte Profiling Document

The multi analyte profiling document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the multi-analyte-profiling technique is performed. In this framework the multi-analyte-profiling is performed for each sample processed by the instrument. As multi-analyte-profiling analyzers are usually plate-based instruments, each well of the plate should be recorded in a separate multi analyte profiling document.

Within the context of the Bio-Plex Manager XML export, the <Well></Well> element is largely analogous to the  multi analyte profiling document in the ASM file.

The location identifier is provided by the RowNo and the ColNo attributes within the <Well> tag. However, in order to report this as single string, these attributes should be concatenated with the RowNo converted to the corresponding letter (1 = A, 2 = B, 3 = C, etc) such that <Well RowNo=”1” ColNo=”1”> would be reported as location identifier: “A1”

The information to be recorded as the description, sample identifier, and sample role type within the sample document does not come from the <Well> node but the <Samples> node within the Bio-Plex Manager XML. In order to correctly associate the data reported within the <Well> node with these properties from the <Sample> node, the RowNo and the ColNo attributes within the <Well> tag must be matched with the same attributes within the <MemberWell> tag within the <Samples> node. Since the <MemberWell> appears lower in the hierarchy, these attributes must be matched and then tags within the parent nodes referenced to obtain the details for  description, sample identifier, and sample role type.

The sample role type is derived from the type tag used within the <Sample> node. These types map to the enumerated sample role types used within the ASM based on the table below.

<Samples> type

ASM sample role type

Blank

blank role

Control

control sample role

Standard

standard sample role

Unknown

unknown sample role

The description and sample identifier should be taken from the <Description> and <Label> tags reported under each instance of a ‘type’ within the <Samples> node.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

The analyte name is reported within the <AnalyteName> tag, nested as a property within the <Samples> node. In order to associate the appropriate analyte name with the other fields in the analyte document, it is first necessary to identify the <MemberWell> using the RowNo and ColNo attributes, then identify the <MWAnalyte> using the RegionNumber attribute.  

Note, there are cases in the XML export in which there is data for assay beads listed within the <BeadRegions> yet no corresponding analyte information within the <MWAnalytes> node. Data should only be returned in the ASM for those cases in which both assay bead and analyte information is present. Those assay beads for which all required data is present are listed within the <RegionsOfInterests> node, nested under <Well> → <RunSettings> → <RegionsOfInterest>.

The error document is an array of objects, each recording details about a specific error.

Calculated Data Aggregate Document / Calculated Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document contains information derived from a single measurement, while the calculated data document is used to report the output of calculations which uses as inputs multiple measurements from the same or different samples within the experiment.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the file export.

All calculated data documents should be placed within the calculated data aggregate document within the multi analyte profiling aggregate document.

While the Bio-Plex Manager XML export does include the results of calculations performed using measurements collected during the experiment, it is not necessary to capture these within the ASM file at this time. Therefore, the calculated data aggregate and associated child documents are not required at this time.
