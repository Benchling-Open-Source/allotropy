Revision 01

Background Section

The MSD Discovery Workbench adapter is intended to convert .txt/.csv exports from MSD’s Discovery Workbench software and generate a .json file corresponding to the plate-reader Allotrope Simple Model. The Discovery Workbench software is in conjunction with Meso Scale Discovery plate readers (MESO SECTOR S 600MM, MSD Sector Imager 2400, MESO Quickplex SQ 120MM). Meso Scale Discovery readers support electrochemiluminescence-based assays for single and multiplex measurements of analytes within biological samples. The Discovery Workbench adapter is expected to recognize the measurement type used to generate the data and generate the plate-reader ASM .json file using the luminescence measurement schema.

Discovery Workbench Specifications

Can include but not limited to:

  • Measurement modes
  • Single - a single luminescence measurement per well
  • Multiplex - multiple spatially arrayed luminescence measurement per well
  • Plate Type/Formats
  • Compatible with 96 and 384-well formats
  • Export Formats (including file extension, delimiter, software-defaults)
  • .TXT and .CSV files
  • By default, exported text files are saved to the C:\ECLResults\ directory
  • Users can define their own file export location
  • Environmental Controls (temperature, O2, CO2)
  • Camera Operating Temperature: -35C

Discovery Workbench Input File Blocks

The MSD Workbench export file works with a .TXT file with a number of ‘blocks’ which contain relevant measurement and metadata required for the ASM.

The ‘blocks’ follow the pattern of:

        (Repeated for N # of Plates)

  • Plate/Measurement Information
  • Start Line: FileName (Only for first plate if multiple plates in file, second plate on will start at line Stack ID)
  • End Line: Orientation
  • Results Data (Plate Matrix)
  • Processing Data
  • Start Line: Radius
  • End Line: Angle
  • Statistics
  • Start Line: Sector
  • End Line: CV (%)
  • Camera Status
  • Start Line: Binning
  • End Line: Temp. Val.
  • Waveform/Current
  • Start Line: Hold
  • End Line: Peak Vltg

Plate-Reader Document

The plate-reader document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the plate-reader technique is performed. In this framework the plate-reader technique is performed for each plate well (rather than plate) in the instrument software export. This means that information from each well should be recorded in a separate plate-reader document.

Handling Measurement Documents/Location Identifiers in MSD Workbench (Text File)

MSD Workbench plates contain a well->spot index per sample that is measured. In the case of multiple spots recorded for a particular well, this will result in multiple measurements (luminescence values) for each associated spot. For handling of the measurement document in the ASM, multiple measurement documents should be instantiated for each spot within a particular well.  

For recording the location identifier of each measurement document associated with a well, the field should be a concatenation of the well (“A1”) and spot index (“1”) (eg. “A1_1”). The well can be determined in the instrument file by the row/column pairing where row is an alphabetical string and column is numeric value. For determining spot index, this will be handled by the number of rows that are within a well row (eg. “A”) until the next alphabetical value is reached (eg. “B”). See image below for reference.

Handling well plate identifier in MSD Workbench (Text File)

MSD Workbench lists up to 3 barcode labels that a user can associate with their well plate. For this version of the adapter, we will only handle parsing the first barcode (Barcode1) as the well plate identifier.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Processed Data Aggregate/Processed Data Document

The processed data document contains results data which is derived from measurement data with values that are dependent on user defined parameters or transformation. The parameters which determine the results within the processed data document are recorded in the data source document.

The MSD Workbench file export does not include results derived from processing sample measurements and therefore there are no records to capture within a processed data document.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the instrument software file export.

All calculated data documents should be placed within the calculated data aggregate document within the plate reader aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same plate reader document, or from measurement documents from different plate reader documents.
