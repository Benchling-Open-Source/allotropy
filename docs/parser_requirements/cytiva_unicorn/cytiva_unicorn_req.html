Cytiva Unicorn High Performance Liquid Chromatography (HPLC) Software Adapter Requirements

Revision # 2

Date: Apr 10, 2025

Background Section

  • Adapter is intended to convert .zip exports from instrument software and generate a .json file corresponding to the liquid-chromatography Allotrope Simple Model
  • Cytiva Unicorn software has the capability to automate chromatography experiment planning, executing and data analysis.
  • Notable HPLC systems that are compatible with Unicorn software are:
  • Akta Avant Chromatography System
  • Akta Go Chromatography System
  • Akta Pure Chromatography System
  • Akta Start Chromatography System
  • The adapter is expected to recognize the detection modality used to generate the data and generate the liquid chromatography ASM .json file using the appropriate measurement schema.

Measurement Document (Core)

The measurement document is an array of objects, each containing details about individual measurements recorded from each well within the instrument software export.

Note:

A total of nine measurement documents should be created for the following (see .json file for reference). This should match the overall structure that was laid out at the top of this document.

  1. Absorbance (UV1) - 1 data cube
  2. Absorbance (UV2) - 1 data cube
  3. Absorbance (UV3) - 1 data cube
  4. Conductivity - 2 data cubes
  5. pH - 1 data cube
  6. Concentration B - 1 data cube
  7. Pressure - 5 data cubes
  8. Flow - 5 data cubes
  9. Temperature - 1 data cube

Some of these fields will only contain one data cube, like Absorbance(UV1), while others contain several data cubes, like Pressure.

The data cubes for Absorbance, pH and Conductivity are generally located in the measurements document (Conductivity also has a cube in processed data doc) as they are considered as active measurements.

The data cubes for Concentration B, Pressure, Flow and Temperature are generally located in either the processed data document or the device control document as they are considered user defined measurements where the user can input a range for the values to be in.

Some of the data cubes might not contain any information (blank or 0).

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the instrument software file export.

All calculated data documents should be placed within the calculated data aggregate document within the liquid chromatography aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same liquid chromatography document, or from measurement documents from different liquid chromatography documents.

Note to Author: For questions regarding the use of a Calculated Data Document within the context of a liquid chromatography adapter requirements - please post questions to #benchling_connect_allotrope_helpers
