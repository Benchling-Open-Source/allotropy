Vi-Cell XR Cell Counter Adapter Requirements

Revision: 05

Date: 2023-09-06

The Vi-Cell XR Cell Counter adapter is intended to convert text file exports from Beckman Vi-Cell XR cell counters to an Allotrope Simple Model (ASM) file for cell counters.

The adapter is expected to convert the exported .xlsx file to an Allotrope ASM .json file

The adapter is expected to produce the following output based on the provided input:

  • Input: Beckman_Vi-Cell-XR_example01_instrumentOutput_PUBLIC.xlsx
  • Output: Beckman_Vi-Cell-XR_example01_asm_rev03.json

  • Input: Beckman_Vi-Cell-XR_example02_instrumentOutput_PUBLIC.xls
  • Output: Beckman_Vi-Cell-XR_example02_asm_rev03.json

Note: For demonstration purposes the output examples only contains the first (3) samples present in the input examples

Additional instrument output file examples:

  • Beckman_Vi-Cell-XR_example03_instrumentOutput_PUBLIC.xls
  • Beckman_Vi-Cell-XR_example04_instrumentOutput_PUBLIC.xlsx
  • Beckman_Vi-Cell-XR_example05_instrumentOutput_PUBLIC.xlsx

Required User Actions

The required user actions are based instructions provided in the Vi-Cell XR Instructions for Use (PN 383674BH, May 2022).

The user is expected to enter Sample ID values within the Vi-Cell XR UI that conform to entity identifiers within Benchling, either existing entity IDs or entities they wish to register upon processing of results.

The processing of a single sample is referred to as a Run within the Vi-Cell XR software. The user is expected to have the sample measurement added to a multi-run file. This is done by either selecting ‘Add to multi-run file’ within the Log in Sample UI or within the Run Options tab in under Preferences (shown below).

This multi-run file should be placed within the in a target directory which is specified during Connection configuration, when creating a Vi-Cell XR Connection within the Benchling UI.

The user is expected to configure the parameters to be included in the multi-run file by selecting the available options within the Export tab under the Preferences (shown below). However, of the available options, only those listed are captured within the ASM data model. Note that the fields that are Required for adapter function

Parameters captured within ASM:

  • Sample ID (Required)
  • Cell Type
  • Minimum Diameter
  • Maximum Diameter
  • Dilution Factor
  • Sample Date
  • Total Cell Count (Required)
  • Viable Cell Count
  • Viability (Required)
  • Total Cells /mL
  • Viable Cells /mL
  • Average Diameter
  • Average Circularity
  • Images

If the user desires to capture the serial number of the instrument used to perform the analysis, this needs to be populated within the Hardware tab under the Configuration menu (shown below).

Generation of the ASM file

The Allotrope ASM for Cell Counters has the following structure

Cell Counting Aggregate Document

⮑ Device System Document

⮑ Data System Document

⮑ Cell Counting Document

⮑ Measurement Aggregate Document

⮑ Measurement Document(s)

⮑ Sample Document

⮑ Device Control Aggregate Document

⮑ Device Control Document(s)

⮑ Processed Data Document

⮑ Data Processing Document

Cell Counting Aggregate Document

The Cell Counting Aggregate Document will contain the Device System Document and the Cell Counting Document.

Device System Document

The device system document within the ASM is intended to capture metadata about the instrument from which the data was obtained.

The Vi-Cell XR .xlsx export only includes minimal device information. The .xlsx should include device serial number within the header portion.

The model number should be recorded as ‘Vi-Cell XR, while the device identifier and parameter can be excluded from the ASM file.

Data System Document

The data system document within the ASM is intended to capture information about the instrument file, its location, source of generation, the adapter/converter used to generate the ASM file.

The data system instance identifier is intended to reference the computer/workstation/file-store on which the instrument file resides. This information is not included in the Vi-Cell XR export, but should be obtained by the adapter querying the identifier of the local system on which it is installed.

The file name is the name of the instrument file which the adapter is using as input.

The UNC path should be the complete file path of the instrument file. The Vi-Cell XR can include a file path in the header portion. However, this may not coincide with the location the adapter picks-up the file, therefore the adapter should record the path of the ‘pick-up’ location.

The software name and software version refer to the application that generated the instrument file. This information is typically included in the Vi-Cell XR export header, therefore ‘Vi-Cell XR’ will be the software name while software version will be the corresponding version (e.g. 2.06.3).

The ASM converter name and ASM converter version refers to the the adapter being described in this requirements document used generate the ASM file from the instrument file. Therefore, the adapter should populate these fields with its own name and version.

Cell Counting Document

The Cell Counting Document is instance of a technique document in the Allotrope parlance, and meant to represent each time the cell counting technique is perform. In this framework the cell counting technique is performed for each sample; the Vi-Cell XR .xslx report is structured with a single sample per line, therefore, each line in the Vi-Cell XR (excluding the header portion and column headers) will correspond to a Cell Counting Document in the ASM.

The Cell Counting Document contains a Measurement Aggregate Document details about analyst - the individual who executed the procedure, and the submitter - the individual who requested the procedure and submitted samples to the analyst.

The Vi-Cell XR does not include the option to report an operator/analyst within the exports file. However, as analyst is a required field in the ASM, a default value of ‘Vi-Cell XR’ should be recorded here.

Measurement Aggregate Document

Measurement Document

As the Vi-Cell XR only performs one type of measurement, brightfield imaging, there will only be a single Measurement Document per Cell Counting Document. Other models of cell counters capable of performing brightfield and fluorescence measurements could have multiple Measurement Documents.

The measurement identifier is not derived from the instrument file and instead should be a universal unique identifier generated and added to the ASM file.

Measurement time should be obtained from the ‘Sample Date’ field within the Vi-Cell XR output. In the ASM measurement time should be recording in a YYYY-MM-DD HH:MM:SS format, were time is formatted to 24-hour, inclusion of T and Z designations can be optionally included if present, otherwise the timezone should be expressed as provided.

Sample Document

The entry defined under Sample ID should be recorded in the sample identifier field in the sample document.

The Vi-Cell BLU sample results export does not include information pertaining to batch identifier, sample role type, or local identifier so these parameters can be excluded from the ASM file.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was perform. Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

As a class of instruments, cell counters will modeled as the core device - cell counter - used to perform the sample handling, and the detector(s) - brightfield and/or fluorescence imagers used to capture images of cells for processing.

As the Vi-Cell XR is a cell counter, the ASM should include a device control document with the device type of ‘cell counter’. The sample volumne setting is the only control parameter to be include within the ‘cell counter’ device control document however this is not included in the Vi-Cell XR export and thus can be excluded.

The Vi-Cell XR instrument relies on brightfield imaging for measuring cells within the samples being processed. Therefore, the device type parameter should always be populated with ‘brightfield imager (cell counter)’ and the detection type parameter with ‘brightfield’.

Similarly, while illumination, exposure and gain are relevant to brightfield imaging, details about these settings are not included, so illumination setting, exposure duration setting, and detector gain setting can also be excluded from the ASM file.

For other models of cell counters utilizing fluorescence imaging, these would include an additional device control document with the device type parameter should always be populated with ‘fluorescence imager (cell counter)’ and the detection type parameter with ‘fluorescence’, illumination, exposure, and detector gain settings; plus additional parameters of excitation wavelength setting, emission wavelength setting , and fluorescence tag. However, since the Vi-Cell XR lacks fluorescence detection, parameters can be excluded from the ASM file.

Processed Data Document

The processed data document contains results data generated by the cell counter. These results are classified as processed data since the values are dependent on which objects are classified as ‘cells’ by the instrument. The parameters which determine the results within the processed data document are recorded within the data processing document.

Data Processing Document

The following column headers within the Vi-Cell XR .xlsx export map to the corresponding field within the data processing document. However, the user must configure the export file to include these parameters (discussed above). If these are not present, then these fields can be excluded from the ASM file.

Image Aggregate Document / Image Document

The Vi-Cell XR export can include file paths to the images collected during the measurement of each sample. These would record within an image aggregate document, with an image document per image file containing details about the index of the image and the filepath recorded as the experimental data identifier.

Alternative Measurement Document

Additionally, as the Vi-Cell XR does not support fluorescence detection, the following fields represented in the alternative form of the measurement document are not present, including:

  • fluorescence tag - captured within device control document
  • fluorescence tag positive cell count - captured within processed data document
  • fluorescence tag positive cell density - captured within processed data document
  • fluorescence tag positive cell percentage - captured within processed data document
