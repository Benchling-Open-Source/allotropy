Roche Cedex HiRes Cell Counting Adapter Requirements

Revision: 01

Date: 2024-07-04

Background Section

  • Adapter is intended to convert CSV/Excel exports from instrument software and generate a .json file corresponding to the cell counting Allotrope Simple Model.
  • Cedex software is used for the Cedex HiRes instrument.
  • The adapter is expected to parse data from the instrument and structure the data from each sample to the associated result fields.

Example I/O Files to ASM JSON

The adapter is expected to produce the following output based on the provided input:

  • Input: roche_cedex_hires_example.xlsx
  • Output: roche_cedex_hires_example.json

Generation of the ASM file

The Allotrope cell counting ASM for Roche Cedex HiRes data has the following structure:

Cell Counting Aggregate Document 1 total

⮑ Device System Document 1 total

⮑ Data System Document 1 total

⮑ Cell Counting Document 1 per sample

⮑ Measurement Aggregate Document 1 per sample

⮑ Measurement Document(s) 1 per sample

⮑ Sample Document 1 per sample

⮑ Custom Information Document 1 per sample

⮑ Device Control Aggregate Document 1 per sample

⮑ Device Control Document(s) 1 per sample

⮑ Processed Data Aggregate Document 1 per sample

⮑ Processed Data Document 1 per sample

⮑ Data Processing Document 1 per sample

⮑ Custom Information Document 1 per sample

ASM schema are detailed in the following:

  • https://github.com/Benchling-Open-Source/allotropy/blob/main/src/allotropy/allotrope/schemas/adm/cell-counting/BENCHLING/2023/11/cell-counting.schema.json

Cell Counting Document

The cell counting document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the cell counting technique is performed. In this framework the cell counting technique is performed for each sample in the instrument software export. This means that information from each sample should be recorded in a separate cell counting document.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Processed Data Aggregate Document / Processed Data Document

The processed data document contains records of information derived from the measurement data itself. As cell counting instruments are typically imaging-based instruments which rely on the processing of images to determine properties such as “cell count” or “viability”, these values are modeled as processed data. The processed data document is captured within a processed data aggregate document, to allow for recording of multiple processed data documents.

  Key Considerations:

  1. File formats supported: .xlsx, .csv. The adapter will read the data from the CSV file/first sheet of the Excel file.
  2. To ensure the adapter functions correctly, the instrument export file must contain following column names in the instrument export file. The columns are as follows:
  1. Cedex ID
  2. System name
  3. System description
  4. Reactor identifier
  5. Sample identifier
  6. Data set name
  7. Dilution
  8. Sample volume
  9. Cell type name
  10. Sample draw Time
  11. Date finished
  12. Username
  13. Avg Diameter
  14. Avg Area
  15. Avg Perimeter
  16. Avg Segm. Area
  17. Avg Compactness
  18. Viable Cell Count
  19. Dead Cell Count
  20. Total Object Count
  21. Viability
  22. Viable Cell Conc.
  23. Dead Cell Conc.
  24. Total Cell Conc.
  25. Std Dev.
  26. Total Cell Count
  27. Aggregate Rate
  1. To ensure the adapter functions correctly, the instrument export file must contain values for the mandatory keys defined in ASM. The required columns are as follows:
  1. measurement time - Date finished
  2. sample identifier - Sample identifier
  3. viability - Viability
  4. viable cell density - Viable Cell Conc.
  1. In this version, we have added few custom fields which require extension to the ASM in the future:
  1. average area
  2. average perimeter
  3. average segment area
  4. standard deviation
  5. aggregate rate
  6. total object count
  7. average compactness
  8. group identifier
  9. sample draw time
  1. The below mentioned fields are expected to have the same values throughout the column in the instrument export file:
  1. Cedex ID
  2. System name
  3. System description
  1. Currently, “Avg Area” and “Avg Segm Area” are marked “(unitless)”. ASM needs to be updated to include "micrometer square (µm²)," and accordingly, the adapter will also require an update.
