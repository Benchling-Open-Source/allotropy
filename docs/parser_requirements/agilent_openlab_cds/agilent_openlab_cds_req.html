Agilent OpenLab CDS Liquid Chromatography Adapter Requirements

Revision 01

Date: 2024-Dec-10

Background Section

  • Agilent OpenLab CDS adapter is intended to convert .rslt folder exports from instrument software and generate a .json file corresponding to the liquid-chromatography Allotrope Simple Model
  • Agilent OpenLab CDS uses two types of software: Acquisition software for instrument management and Data Analysis software for result interpretation. It supports techniques such as liquid chromatography(LC), gas chromatography(GC), liquid chromatography-mass spectrometry(LC-MS) and gas chromatography-mass spectrometry(GC-MS)
  • The adapter is expected to recognize the detection modality used to generate the data and generate the liquid chromatography ASM .json file using the appropriate measurement schema

Example I/O Files to ASM JSON

The adapter is expected to produce the following output based on the provided input:

Liquid chromatography example:

  • Input: 20241104 Seq2 SEC.rslt
  • Intermediate json: structured_intermediate_json
  • output: 20241104_Seq2_SEC.json

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the instrument software file export.

All calculated data documents should be placed within the calculated data aggregate document within the liquid chromatography aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same liquid chromatography document, or from measurement documents from different liquid chromatography documents.

Key Considerations:

  1. The adapter supports .rslt folder format. It reads the data from .acaml, .dx, .rx, .sqx files inside the folder, converts it into the intermediate structured JSON format, and then maps this intermediate format to the ASM-specific JSON format
  2. The adapter handles only Liquid chromatography technique and single sequence run folder.
  3. The adapter handles both raw data from acquisition software and processed data from data analysis software located in the .rslt folder.
  4. In this version, we have added a few custom fields which might require extension to the ASM in the future:
  1. Device Name
  2. Analytical Method Identifier
  3. Maximum Temperature
  4. Maximum Pressure
  5. Firmware version (inside Chromatography Column Document)
  6. Injection Source
  7. Acquisition Method Identifier
  8. Injector Position
  9. Location Identifier (inside Sample Document)
  10. Replicate (inside Sample Document)
  11. Sample Amount (inside Sample Document)
  12. Barcode (inside Sample Document)
  13. Emission Wavelength Setting
  1. To ensure the adapter functions correctly, the instrument export file must contain values for the mandatory keys defined in ASM. The required fields are as follows:
  1. Autosampler Injection Volume Setting(chromatography) - Result Data.Sample Data.InjectionVolume
  2. Injection Identifier - Result Data.Sample Data.Sample Measurement Data.InjectionMeasData_ID
  3. Injection time - Result Data.Sample Data.RunDateTime
  4. Sample Identifier - Result Data.Sample Data.SampleName
  5. Device type - Metadata.Instrument.Module.Type
