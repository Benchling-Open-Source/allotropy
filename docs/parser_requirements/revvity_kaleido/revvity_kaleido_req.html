Revvity Kaleido Plate Reader Adapter Requirements

Revision 08

Date: 2024-03-05        

The Revvity Kaleido Plate Reader adapter is intended to convert CSV exports from Kaleido software and generate a .json file corresponding to the plate-reader Allotrope Simple Model (ASM). The Kaleido software is used in conjunction with plate readers manufactured by Revvity, such as the EnSight multi-mode plate reader/imager. These instruments are capable of performing absorbance, fluorescence, luminescence measurements as well as optical-imaging. The plate-reader ASM supports the recording of data from these different measurement modalities by referencing separate measurement schema for uv-absorbance, fluorescence, luminescence or optical-imaging. The adapter is expected to recognize the detection modality used to generate the data and generate the plate-reader ASM .json file using the appropriate measurement schema.

The adapter is expected to produce the following output based on the provided input:

Alpha assay (Fluorescence) example:

  • Input: Revvity_Kaleido_Alpha_Example1.csv
  • Output: Revvity_Kaleido_Alpha_Example1.json 

Absorbance assay example:

  • Input: Revvity_Kaleido_ABS_Example1.csv
  • Output: Revvity_Kaleido_ABS_Example1.json 

Fluorescence Intensity example:

  • Input: Revvity_Kaleido_FI_Example1.csv
  • Output: Revvity_Kaleido_FI_Example1.json 

Additional example file:

Revvity_Kaleido_FI_Example2.csv

Luminescence assay example:

  • Input: Revvity_Kaleido_Lum_Example1.csv
  • Output: Revvity_Kaleido_Lum_Example1.json 

Additional example files:

Revvity_Kaleido_Lum_Example2.csv

Revvity_Kaleido_Lum_Example3.csv

Optical-Imaging example:

  • Input: Revvity_Kaleido_IMG_Example1.csv
  • Output: Revvity_Kaleido_IMG_Example1.json

Note: for demonstration purposes the output examples only contain the first four wells of the sample plate present in the input examples

The ASM model schema are detailed in the following:

  • plate-reader ASM schema
  • ultraviolet-absorbance-point-detection ASM schema
  • fluorescence-point-detection ASM schema
  • luminescence-point-detection ASM schema

Kaleido User Settings

Measurement Modes:

Measurement mode corresponds to the options a user can select for the type of measurement operation that the reader will perform–determines the number of measuring events that happens per well

  • Single (one detection wavelength, one timepoint)
  • Kinetic (one detection wavelength, multiple timepoints)
  • Well-Scan (one detection wavelength, multiple positions in the well)
  • Wavelength Scan (multiple detection wavelengths)

Note: Currently the ASM only supports ‘Single’ endpoint reads. This adapter will only be expected to support reads of this type for Kaleido output files.

Plate Types:

Type of plate formats that Kaleido is compatible with–based on number of wells in the plate

  • 6, 12, 24, 48, 96 and 384-well
  • Imaging can only be used with 96 and 384 well plates

Export Formats:

Formatting and file extension of Kaleido software output options from plate reads. Details of plate, measurement, and sample information along with raw/processed data contained within

 

  • CSV_PLATE (Default)
  • CSV_LIST
  • XML
  • CUSTOM
  • ARCHIVE

Note: Currently the export file formats that this adapter will be expected to support are of CSV_PLATE format. Example files above are all of CSV_PLATE format.

Plate-Reader Aggregate Document

The plate-reader aggregate document represents the highest level of organization in the Plate-Reader ASM, and will contain the device system document - for recording details about the instrument, the data system document - for recording details about the data file, the plate-reader document - for recording data from each well of the plate, and where appropriate a calculated data aggregate document for recording calculated results derived from multiple plate-reader documents (e.g. aggregated results from multiple wells).

Device System Document

The device system document within the ASM is intended to capture metadata about the instrument from which the data was obtained.

The fields within this document can include:

  • asset management identifier
  • description
  • brand name
  • product manufacturer
  • device identifier (required)
  • model number (required)
  • equipment serial number
  • firmware version

For device identifier and model number, because these are not expliciting listed in the file, use EnSight as the default value. Note: This may change with additional example files

For equipment serial number, use the Instrument Serial Number field found in the Measurement Basic Information (Kaleido v2.0+)/Instrument Information block (Kaleido v3.0+).

product manufacturer is not explicitly listed in the file, this should be as default “Revvity”.

The additional parameters in the device system document (asset management identifier, description, brand name, firmware version) are not listed in the example file and can be left not recorded.

Data System Document

The data system document within the ASM is intended to capture information about the instrument file, its location, source of generation, the adapter/converter used to generate the ASM file.

The fields within this document can include:

  • data system instance identifier
  • file name
  • UNC path
  • software name
  • software version
  • ASM converter name
  • ASM converter version

The data system instance identifier is intended to reference the computer/workstation/file-store on which the instrument file resides. This information is not included in the export, but if possible should be obtained by the adapter querying the identifier of the local system on which it is installed.

The file name is the name of the instrument file which the adapter is using as input.

The UNC path should be the complete file path of the instrument file.

The software name and software version refer to the application that generated the instrument file. For the Kaleido, this information is provided on the last line of the .csv export, where the software name - Kaleido, and software version is reported. (same logic as EnVision adapter)

Example: “Exported with Kaleido version 3.0.3067.117”

The ASM converter name and ASM converter version refers to the adapter being described in this requirements document used to generate the ASM file from the instrument file. Therefore, the adapter should populate these fields with its own name and version.

Plate-Reader Document

The plate-reader document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the plate-reader technique is performed. In this framework the plate-reader technique is performed for each plate well (rather than plate) in the Kaleido export. This means that information from each well should be recorded in a separate plate-reader document.

The fields within this document can include:

  • measurement aggregate document (required)

Measurement Aggregate Document

The measurement aggregate document contains records of the measurements performed, each captured in a measurement document as metadata about the measurements performed. Note that the measurement aggregate document and constituent documents (e.g. measurement document, sample document) within the Plate-Reader ASM inherits generic fields from the core ASM schema as well as specific fields from the plate-reader ASM schema.

The fields within this document can include:

  • measurement time (required)
  • analytical method identifier
  • experimental method identifier
  • experiment type
  • container type
  • plate well count (required)
  • well volume
  • measurement document (required - per core schema)
  • image aggregate document

For container type, “well plate” can be used as the default value.

For plate well count, reference the Number of Rows field and Number of Columns field. Perform calculation of (Number of Rows) x (Number of Columns) for plate well count.

For measurement time, reference the “Measurement Started” field in the Measurement Basic Information block (Kaleido v2.0+)/Measurement Information block (Kaleido v3.0+). The format should be reformatted to DateTime to match the measurement time data type.

For experiment type, reference the Background Information/Results block, “Result(s) from <experiment type> 1” field. The <experiment type> should be the recorded value in the ASM experiment type parameter

Measurement Document

The measurement document is an array of objects, each containing details about individual measurements recorded from each well within the Kaleido export.

The fields within this document can include:

  • measurement identifier (required)
  • sample document (required - per core schema)
  • device control aggregate document (required - per core schema)
  • compartment temperature (required)
  • measurement fields from one of:
  • ultraviolet-absorbance schema
  • fluorescence schema
  • luminescence schema
  • optical-imaging schema
  • image aggregate document

The measurement identifier is not derived from the instrument file and instead should be a universal unique identifier generated and added to the ASM file.

The compartment temperature is not expressly reported in the Kaleido output files. Thus this field should be left unrecorded.

Measurement field Units

  • From ultraviolet-absorbance schema the absorbance field should be reported in “mAU” units.
  • From fluorescence schema the fluorescence field units should be reported in RFU for Alpha and Fluorescence Intensity experiments
  • From luminescence schema the luminescence field should be reported in “RLU” units

Sample Document

The sample document contains details of the sample being assessed as part of the measurement.

The fields within this document can include:

  • description
  • sample identifier (required)
  • batch identifier
  • sample role type
  • written name
  • group identifier
  • location identifier (required) - one of:
  • well location identifier
  • vial location identifier
  • well plate identifier
  • mass concentration

The Kaleido output does not include records that correspond to sample description, batch identifier,  written name, or mass concentration.

The sample identifier corresponds to the string present in the PlateMap block that matches to the same location identifier. If this string is not present then sample identifier should be recorded as a concatenation of well plate identifier, location identifier (eg. <well plate identifier>_<location identifier>).

The well plate identifier corresponds to the plate barcode.

In Kaleido v2.0+, the barcode value is reported in the same cell as the field name (eg. "Barcode: V-2020-10-26 16:16:17").

In Kaleido v3.0+, the barcode value is reported in the cell below the field name.

This field can be found in the Background Information/Results block. If this field is empty, this field can be left unrecorded. Appears in some example files as value in Measurement Date field (measurement time). Usually due to error in user scan of barcode or no barcode present.

The location identifier needs to be inferred from the position of the measurement within the Background Information/Results block. Concatenation of the row (A, B, C...) and column (1, 2, 3...) identifiers should be recorded (Ex. “A1”, “H6”).

The sample role type is reported within the corresponding position of as well location identifier but within the Platemap block. Kaleido uses the following abbreviations for role types:

Image Aggregate Document

The image aggregate document contains records of the image files recorded from the instrument per well, each captured in an image document which will contain results data and metadata of the raw image files obtained.

The image aggregate document inherits its schema from the optical-imaging ASM schema.

image aggregate document can be a child document of the measurement aggregate document when the image file recorded cannot be determined to which measurement document it came from either by the file name or within the instrument file. If the image file’s associated measurement document items can be discerned, then the image aggregate document should be associated as a child of the measurement document.

For Kaleido, default handling will be to record the image aggregate document/image document as a child of the measurement aggregate document since the device control parameters associated with an image are not specified in the instrument file/image file name.

The fields within this document can include:

  • image document (required)

Image Document

The image document is an array of objects, each containing details about individual raw images recorded from each well within the Kaleido export.

The fields within this document can include:

  • index
  • experimental data identifier (required)

The experimental data identifier is derived from the instrument file in the Background Information/Results block and corresponds to the image file name. The identifier will map to the location identifier in the sample document of the measurement documents in the measurement aggregate document.

Handling of multiple images will be done by adding additional image documents to the image aggregate document in the measurement aggregate document.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

For a multi-mode plate reader such as the Kaleido there will be an entry in the device control document for the detection module: fluorescence, or luminescence. Additional entries to the device control document could be made for: the shaking configuration of the plate carrier, an integrated plate stacker, a gas control module, and/or a reagent injection module - however, details about these other module types are not included within the Kaleido export.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

The fields within this document can include:

  • device identifier
  • device type (required)
  • detection type
  • product manufacturer
  • brand name
  • equipment serial number
  • model number
  • firmware version
  • shaking configuration description - used within a device control document for carrier shaking
  • detector distance setting (plate reader)
  • integration time
  • number of averages
  • detector gain setting
  • scan position setting (plate reader)
  • detector carriage speed setting
  • device control fields from one of:
  • ultraviolet-absorbance schema
  • detector wavelength setting
  • detector bandwidth setting
  • fluorescence schema
  • detector wavelength setting
  • detector bandwidth setting
  • wavelength filter cutoff setting
  • excitation wavelength setting
  • excitation bandwidth setting
  • luminescence schema
  • detector wavelength setting
  • detector bandwidth setting
  • optical-imaging schema
  • detector wavelength setting
  • excitation wavelength setting
  • magnification setting
  • exposure duration setting
  • illumination setting
  • transmitted light setting
  • auto focus setting
  • image count setting
  • fluorescent tag setting

/luminescence

For measurements performed with the ‘base’ Kaleido instrument, a number of the fields within the device control document can be ignored, since this information has already been captured within the device system document. These fields include: device identifier, product manufacturer, brand name, equipment serial number, model number, firmware version.

Based on software version Kaleido has two fields that list the acquisition mode:

  • Kaleido v2.0: “Tech” field
  • Kaleido v3.0+:  “Operation” field

In order to assign a detection type - measurements corresponding to luminescence measurements will include the string “Luminescence”. Labels that correspond to fluorescence measurements will include strings such as: “Alpha” or “Fluorescence intensity”. Labels that correspond to absorbance measurement will include strings such as: “ABS”. Labels that correspond to optical-imaging measurements will include strings such as “IMG”.

For device type, “plate reader” can be used as the default value. Based on the type of measurement from the detection type, the value can be replaced with “fluorescence detector”, “luminescence detector”, “absorbance detector”, “imaging detector”.

For  number of averages, use the “Number of Flashes” field found in the Measurement (Kaleido v2.0+)/Details of Measurement (Kaleido v3.0+) block. Note that this field will not be present in all files types including luminescence, Alpha, imaging reads

For luminescence, absorbance, and fluorescence files detector distance setting (plate reader), use the “Distance between Plate and Detector” field found in Measurement (Kaleido v2.0+)/Details of Measurement (Kaleido v3.0+) block. For optical-imaging files, use the “Focus height [µm]” field found in Measurement (Kaleido v2.0+)/Details of Measurement (Kaleido v3.0+) block.

For  scan position setting (plate reader), use the “Excitation/Emission” field found in Measurements (Kaleido v2.0+)/Details of Measurement (Kaleido v3.0+) block. *Note: this field will be present in Fluorescent Intensity experiments

The mode specific measurement: absorbance, fluorescence or luminescence is used depending on the acquisition modality used.

For absorbance, detector wavelength setting can be found  as the “Excitation Wavelength [nm]” field.

For fluorescence, detector wavelength setting can be found as the “Emission Wavelength [nm]” field, and for fluorescence and optical-imaging the excitation wavelength setting can be found as the “Excitation Wavelength [nm]” field.

For magnification setting, the default magnification for the Kaleido is 4x and can be recorded as value of 4 with unitless units.

exposure duration setting can be found as the “Exposure time [ms]” field in the Measurement (Kaleido v2.0+)/Details of Measurement (Kaleido v3.0+) block.

Enumerated list: [“brightfield”, “darkfield”, “phase contrast”]. BRIGHTFIELD maps to “brightfield”, if field “Generate DPC” is TRUE, then maps to “phase contrast”.

fluorescent tag setting can be derived from the “Channel” field in the Measurement (Kaleido v2.0+)/Details of Measurement (Kaleido v3.0+) block. If “BRIGHTFIELD” is listed in the “Channel” field, then this parameter should be left unrecorded.

Processed Data Aggregate/Processed Data Document

The processed data document contains results data which is derived from measurement data with values that are dependent on user defined parameters or transformation. The parameters which determine the results within the processed data document are recorded in the data source document. Processed image features/analyses will be recorded in the image feature document.

The fields within this document can include:

  • image feature aggregate document
  • image feature document (required)
  • image feature identifier (required)
  • image feature name
  • image feature result
  • data source aggregate document
  • data source document
  • data source identifier (required)
  • data source feature 

Image Feature Aggregate/Image Feature Document

The image feature document contains processed image analysis results data which is derived from data source document(s) on a per well basis rather than an aggregation between wells/samples. The image feature document fields record user defined parameters or transformations on image(s)

The fields within this document can include:

  • image feature identifier (required)
  • image feature name
  • image feature result

image feature identifier should be recorded by a UUID for each separate image feature document.

Processed image feature data will be formatted the same as the Results block with a column in the block labeled “AnalysisParameter: (image feature name)” and the resulting data in a plate matrix below.

image feature name field should be recorded from the AnalysisParameter field.

image feature result field should be recorded from the plate matrix, mapped from the sample document->location identifier. For recording the units of the image feature result, unless explicitly listed in the image feature name with square brackets (Ex: “[%]”), record the units of the image feature result to be “unitless”.

Each image feature document should have a correspondent data source document where the data source identifier should be recorded as the image file name from which the image features comes from.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the Kaleido file export.

All calculated data documents should be placed within the calculated data aggregate document within the plate reader aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same plate reader document, or from measurement documents from different plate reader documents.

The fields within this document can include:

  • calculated data identifier
  • data source aggregate document
  • data source document
  • data source identifier
  • data source feature
  • calculated data name
  • calculation description
  • calculated result

Although calculated data documents are built into this adapter requirements, no example files contain data that should be recorded in calculated data documents as of this revision.
