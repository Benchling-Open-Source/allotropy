BD Biosciences FACSDiva Flow Cytometry Adapter Requirements

Revision 01

Date: 2025-03-28

The BD Biosciences FACSDiva adapter is intended to convert XML  exports from BD Biosciences FACSDiva software and generate a .json file corresponding to the flow cytometry Allotrope Simple Model. The FACSDiva software is an analysis software used in conjunction with flow cytometers manufactured by BD Biosciences, including FACSCanto, FACSAria, LSRFortessa, LSRCelesta. The FACSDiva software supports single-cell flow cytometry analysis that helps you interpret your data quickly and effectively with accessible features for immunophenotyping, cell cycle analysis, proliferation, kinetics studies, quantitative population comparison, high-dimensional analysis, plate screening assays. The BD Biosciences FACSDiva adapter is expected to recognize the analysis used to generate the data and generate the flow cytometry ASM .json file. Additionally associated with the XML file exported from FACSDiva, are standard FCS files that will be generated by the cytometer software prior to analysis.

Flow Cytometry Document

The flow cytometry document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the flow cytometry technique is performed. In this framework the flow cytometry technique is performed for each sample in the instrument software export. This means that information from each sample should be recorded in a separate flow cytometry document.

Error Aggregate Document / Error Document

The error document within the ASM is intended to record information about any errors returned by the instrument in the course of performing the measurement. As multiple errors may be reported per measurement, these are structured within an error aggregate document which in turn contains the error document. The error document is an array of objects, each recording details about a specific error.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Processed Data Aggregate Document / Processed Data Document

The processed data document contains records of information derived from the measurement data itself. As flow cytometry instruments typically include analyses which rely on the processing of populations/gating to determine population counts/statistics, these values are modeled as processed data. The processed data document is captured within a processed data aggregate document, to allow for recording of multiple processed data documents.

Statistics Document

The statistics document is a enumerated list of statistical features which will contain values of statistical parameters on populations.

For determining which population that statistics will be derived from, the statistics will appear as a subpopulation of the statistic that they determined for.
