Thermo Fisher Scientific Genesys 30 Spectrophotometry Adapter Requirements

Revision 03

Date: 2024-08-02

Background Section

  • Adapter is intended to convert .csv or .tsv exports from instrument software and generate a .json file corresponding to the spectrophotometry Allotrope Simple Model.
  • Inbuilt Genesys 30 instrument software is used for the Thermo Fisher Scientific Genesys 30 Spectrophotometry instrument.
  • The adapter is expected to parse the data and generate the spectrophotometry ASM JSON file using the absorbance measurement schema.

The adapter is expected to produce the following output based on the provided input:

Spectrophotometry Aggregate Document

The spectrophotometry aggregate document represents the highest level of organization in the spectrophotometry ASM, and will contain the device system document - for recording details about the instrument, the data system document - for recording details about the data file, the spectrophotometry document - for recording data from each well of the plate, and where appropriate a calculated data aggregate document for recording calculated results derived from multiple spectrophotometry documents (e.g. aggregated results from multiple wells).

Spectrophotometry Document

The spectrophotometry document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the spectrophotometry technique is performed. In this framework the spectrophotometry technique is performed for each sample and/or plate well (rather than plate) in the instrument software export. This means that information from each well should be recorded in a separate spectrophotometry document.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control documents. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Calculated Data Aggregate / Calculated Data Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document is a facet of a measurement document and contains results derived from transformation of the data within that measurement document.

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the instrument software file export.

All calculated data documents should be placed within the calculated data aggregate document within the spectrophotometry aggregate document, regardless of whether the calculated data is derived from measurement documents from within the same spectrophotometry document, or from measurement documents from different spectrophotometry documents.

Key Considerations:

  1. File formats supported: .csv or .tsv. The adapter will read the data from the CSV or TSV files.

  1. The Genesys 30 is a visible spectrophotometer designed specifically to support measurements within the visible spectrum.

  1. There are three methods in the Genesys 30 which are Scan, Quant and Fixed. Currently, the adapter supports the “Scan” method with ABS mode. Also, for the Scan method, there are two modes ABS (Absorbance) or %T (% Transmittance).

  1. To ensure the adapter functions correctly, the instrument export file must contain following field names in the instrument export file for the Scan method. The fields are as follows:
  1. Scan
  2. Mode
  3. Lower
  4. Upper
  5. Length
  6. wavelength(nm)
  7. ABS (in case of ABS mode)
  8. %T (in case of %T mode)

  1. The attached input file is intended for ABS mode, but it also includes transmittance values. For this ABS mode, the transmittance values have been excluded.

  1. To ensure the adapter functions correctly, the instrument export file must contain values for the mandatory keys defined in ASM. The required fields are as follows:
  1. measurement time - Scan
  2. absorbance - ABS

  1. In this version, we have added few custom fields which require extension to the ASM in the future:
  1. operating minimum
  2. operating maximum

  1. If the units for Lower, Upper are unspecified in the exported file, “nm” will be set as the default unit based on the instrument configuration.

  1. If the units for Absorbance and Transmittance are unspecified in the exported file, “mAU” and “%” will be set as the default unit based on the standard units respectively.

  1. The instrument export file should include the expected export fields specified in point 3, without any additional data.

  1. GENESYS 30 does not support saving Live Display methods to the library because the parameters of the measurement are so simple. For this purpose, VISIONlite™ software  is used to record the   Live Display data electronically. Hence, VISIONlite is not a mandatory software for this instrument but an optional one.
