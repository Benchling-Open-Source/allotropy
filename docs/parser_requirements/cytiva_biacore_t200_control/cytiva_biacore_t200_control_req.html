Cytiva Biacore T200 Control Binding Affinity Analyzer Adapter Requirements

Revision 01

Date: 2024-10-30

Background Section

  • Cytiva Biacore T200 Control adapter is intended to convert .blr exports from instrument software and generate a .json file corresponding to the binding-affinity-analyzer Allotrope Simple Model
  • Cytiva Biacore instruments uses two types of software: Control software namely Biacore Control software and Analysis software namely Biacore Insight Software. This binding affinity analyzer operates using the surface plasmon resonance (SPR) technique.
  • The adapter is expected to recognize the detection modality that is used to generate the data and generate the binding-affinity-analyzer ASM .json file using the appropriate measurement schema.

Example I/O Files to ASM JSON

The adapter is expected to produce the following output based on the provided input:

Surface plasmon resonance binding assay example:

  • Input: kinetics-instrument-file.blr , immobilization-instrument-file.blr
  • Intermediate JSON: kinetics-intermediate-file.json, immobilization-intermediate-json
  • Output: kinetics-asm-output.json, immobilization-asm-output.json

In this framework the binding affinity analysis technique is performed for each assay/run from the instrument software export. This means that information from each assay/run should be recorded in a separate binding affinity analyzer document.

Sample role type enumerated list (see resource here for other ontological definitions):

  • control sample role - A control sample role is a sample role of a material sample that does contain a known amount of analyte for control purposes.

  • standard sample role - A standard sample role is a sample role of a material sample that does contain a known amount of standard.

  • validation sample role - A validation sample role is a sample role of a material sample that is used for validation purposes.

  • experiment sample role - An experiment sample is a role that is played by a material object of interest in an assay used to obtain generalizable information about the sample source.

  • sample role - A sample role type is a classification datum that classifies samples by the type of sample role in the experiment

  • spiked sample role - A spiked sample role is a sample role of a material sample that does contain a spike substance.

  • blank role - A blank role is a sample role of a material that does not contain any analyte.

  • unknown sample role - An unknown sample role is a sample role of a material of an unknown substance, or that does contain an unknown amount of analyte.

  • calibration sample role - A calibration sample role is a sample role of a material that does contain a known amount of substance for calibration purposes.

  • unspiked sample role - An unspiked sample role is a sample role of a material sample that does not contain a spike substance.

  • specimen role - Specimen role is the sample role of a prepared fraction of a portion of material that is analyzed in a particular experiment.

  • quality control sample role - A quality control sample role is a sample role of a material sample that is used for purposes of quality control.

  • reference sample role - A reference sample role is a sample role of a material sample that does contain a known amount of analyte.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Key Considerations:

  1. The adapter supports .blr file format. It reads data from the proprietary .blr file, converts it into an intermediate structured JSON format, and then maps this intermediate format to the ASM-specific JSON format.
  2. The adapter handles only the kinetics and immobilization wizard files.
  3. The adapter handles only the .blr file which contains data from the control software, and doesn’t have any process data from the analysis software.
  4. In this version, we have added few custom fields which might require extension to the ASM in the future:
  1. baseline flow
  2. data collection rate
  3. dip details
  4. ifc identifier
  5. lot number
  6. last modified time
  7. last use time
  8. first dock date
  9. number of flowcells
  10. number of spots
  11. buffer volume
  12. molecular weight
  1. To ensure the adapter functions correctly, the instrument export file must contain values for the mandatory keys defined in ASM. The required fields are as follows:
  1. device identifier - system_information.InstrumentId
  2. device type - application_template_details.Flowcell <index> (in case of immobilization)
  3. Measurement time - system_information.Timestamp
  4. POSIX path - cycle_data.<index>.sensorgram_path
  5. sample identifier - sample_data.<index>.sample_name (in case of kinetics)
  6. Sensor chip identifier - chip.Id
