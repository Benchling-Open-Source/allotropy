Revision 01

Date: 2024-06-05

Background Section

The Roche Cedex BioHT adapter is intended to convert .TXT exports from instrument software and generate a .json file corresponding to the Solution Analyzer Allotrope Simple Model (ASM). The Roche Cedex BioHT is a cell culture analyzer instrument that supports metabolite-detection and optical density measurements of bioprocess samples using spectrophotometric detectors. Measurement results will fall under two categories of measurement schemas from the Solution Analyzer ASM (uv-absorbance point-detection and metabolite-detection). The adapter is expected to recognize the detection modality used to generate the data and generate the solution-analyzer ASM .json file using either uv-absorbance point-detection and metabolite detection schemas.

Note that the Roche Cedex BioHT can perform measurements of both uv-absorbance and metabolite detection on a single sample, thus both schemas can be referenced in the final JSON output. This is discussed further in the documentation below.

Solution-Analyzer Document

The solution analyzer document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the solution-analyzer technique is performed. In this framework the solution-analyzer technique is performed for each sample in the instrument software export. This means that information from each sample should be recorded in a separate solution-analyzer document.

Measurement Aggregate Document

The measurement document is an array of objects, each containing details about individual measurements recorded from each sample within the instrument software export. Note that the measurement aggregate document and constituent documents (e.g. measurement document, sample document) within the Solution-Analyzer ASM inherits generic fields from the core ASM schema as well as specific fields from the solution-analyzer ASM schema. As mentioned above, there are additional extensions to the solution-analyzer ASM schema defined in respective measurement schemas (e.g. measurement document, device control document, analyte document, processed data document) that solution-analyzers support,  including cell-counting, metabolite-detection, osmolality, uv-absorbance, blood-gas-detection, pH detection.

Handling of Measurement Documents in Roche Cedex BioHT

The Roche Cedex BioHT as mentioned above can handle multiple measurement modalities to record measurement results. For recording of measurement documents within the Solution Analyzer ASM, measurement documents should be instantiated for each measurement modality (eg. cell-counting, metabolite-detection, osmolality, uv-absorbance, blood-gas-detection, pH detection) that is recorded for a sample.

For example, 2 measurement documents should be instantiated if a uv-absorbance measurement and metabolite-detection measurement are recorded from the instrument file for a sample with the same sample identifier.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Analyte Aggregate/Analyte Document (Metabolite Detection)

As several analytes are typically measured in these assays, these are structured within an analyte aggregate document which in turn contains the analyte document. Handling Analyte Documents for Roche Cedex BioHT

Multiple analytes can be detected for a given sample (ie. sample identifier). For handling of multiple analyte results performed on one sample, the Roche Cedex BioHT adapter should instantiate one analyte aggregate document and record each of the analytes identified in the instrument file for the sample as separate analyte documents.
