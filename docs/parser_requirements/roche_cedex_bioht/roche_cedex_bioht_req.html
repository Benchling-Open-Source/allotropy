Roche Cedex Bio HT Adapter Requirements

Revision: 06

Date: 2023-06-22

The Roche Cedex adapter is intended to covert text file exports from Roche Cedex Bio HT cell culture analyzers to an Allotrope Simple Model (ASM) file for cell culture analyzers.

The adapter is expected to:

  • Watch a designated file directory for .txt file matching the pattern of Cedex archive files
  • Convert the archive .txt file to an Allotrope ASM .json file
  • Return the original archive .txt file and converted ASM .json file to a designated S3 bucket.

The adapter is expected to produce the following output based on the provided input:

  • Input: Roche_Cedex_BioHT_example01_instrumentOutput.txt
  • Output: Roche_Cedex_BioHT_example01_asm_rev3.json

Additional example

  • Input: Roche_Cedex_BioHT_example04_instrumentOutput.txt
  • Output: Roche_Cedex_BioHT_example04_asm_rev2.json

An additional examples of a Cedex BioHT archive files:

  • Roche_Cedex_BioHT_example02_instrumentOutput.txt
  • Roche_Cedex_BioHT_example03_instrumentOutput.txt

The ASM for cell culture analyzers is detailed in the following

Generation of the ASM file

The Allotrope ASM for Cell Culture Analyzers has the following structure

⮑ Device System Document

⮑ Data System Document

⮑ Cell Culture Analyzer Document(s)

⮑ Measurement Aggregate Document

⮑ Measurement Document(s)

⮑ Sample Document

⮑ Device Control Aggregate Document

⮑ Device Control Document(s)

⮑ Analyte Aggregate Document

⮑ Analyte Document

Cell Culture Analyzer Aggregate Document

The cell culture analyzer aggregate document will contain the device system document , data system document and the cell culture analzyer document(s).

Device System Document

The device system document within the ASM is intended to capture metadata about the instrument from which the data was obtained.

The archive file info row within the Cedex archive file contains entries for several metadata fields within the ASM, including those for the Device System, Data System, and Measurement Aggregate Documents. The archive file info row is indicated within the archive file as the first row and containing a value of (0) in the first column.

The model number is recorded the seventh column of the archive file info row. In the examples, this field contains, ‘CEDEX BIO HT’.

The device serial number is recorded in the eighth column of the archive file info row. In the first example, this field has the value ‘620103’.

The product manufacturer can be recorded as “Roche”.

All other fields within the Device System Document can be ignored.

Data System Document

The data system document within the ASM is intended to capture information about the instrument file, its location, source of generation, the adapter/converter used to generate the ASM file.

The data system instance identifier is intended to reference the computer/workstation/file-store on which the instrument file resides. This information is not included in the BioHT export, but if possible should be obtained by the adapter querying the identifier of the local system on which the Gateway hosting the adapter is installed.

The file name is the name of the instrument file which the adapter is using as input.

The UNC path should be the complete file path of the instrument file.

The software name and software version refer to the application that generated the instrument file. The sofware name is not included in the export file, therefore ‘CEDEX BIO HT’ can be used as the default name, while the software version is recorded in the ninth column of the archive file info row..

The ASM converter name and ASM converter version refers to the the adapter being described in this requirements document used generate the ASM file from the instrument file. Therefore, the adapter should populate these fields with its own name and version.

Cell Culture Analyzer Document

The Cell Culture Analyzer Document is an example of a technique document in the Allotrope parlance, and meant to represent each time a sample is assessed using a cell culture analyzer. The Cedex Bio HT export is structured with a single line per test performed on a sample, therefore, therefore all consecutive lines containing the same Order ID (third column) - OR- combination of Order ID (third column) and Reactor ID (fourth column, optional) will correspond to a Cell Culture Analyzer Document in the ASM.

The Cell Culture Analyzer Document contains a Measurement Aggregate Document, details about analyst - the individual who executed the procedure, and the submitter - the individual who requested the procedure and submitted samples to the analyst.

The analyst field corresponds to the system user recorded in the final (tenth) column of the archive file info row, shown as ‘ADMIN’ in the examples. As there is only a single analyst entry per export file, the analyst will be repeated for all Cell Culture Analyzer Documents in the ASM.

Measurement Aggregate Document

The data processing time corresponds to the timestamp recorded in the second column archive file info row.

All times recorded in the ASM file should be expressed in the timezone when timezone information is provided. Otherwise, the time should reported as expressed within the file.

Measurement Document

The information to be recorded in the measurement document of the ASM file is contained within the Sample rows within the Cedex archive file. The Sample rows within the file are those containing a value of (40) or (50) in the first column.

The Cedex archive files are organized as a single line per analyte/property measurement, however, a single measurement document should be generated per Order ID (third column). The Order ID usually refers to a single biological sample, and data from these instruments it typically analyzed in a ‘sample-centric’ rather than ‘analyte-centric’ fashion.

The measurement identifier is not derived from the instrument file and instead should be a universal unique identifier generated and added to the ASM file.

The measurement time should reflect the Date-Time entry for the first recording for a given sample captured under Order ID. There is only one measurement time per measurement document and while Cedex archive file contains a timestamp for each analyte processed these times are usually within a few seconds of each other for measurements performed on a given sample.

Sample Document

The entry defined under Order ID should be recorded in the sample identifier field in the sample document.

The Cedex archive file includes an optional second identifier referred to as the Reactor ID. This value would correspond to the batch identifier within the ASM. The Reactor ID / batch identifier appears within the fourth column of the archive file.

In a particular Cedex archive .txt file Order ID and Reactor ID may may be used together to denote a unique sample, while specific values of each may be repeated. Therefore, when both Order ID/sample identifier AND Reactor ID/batch identifier are present, these values together should be used to key a separate cell culture analyzer documents (and thus separate a measurement document ) within the ASM.

The sample role type property within the ASM is determined by the Sample Type column (fifth column) within the Cedex archive file. However, the Cedex output universally reports these as ‘Sample’ - abbreviated SAM, and as such this file property can be excluded from the ASM.

Device Control Aggregate / Device Control Doc

The standard format of the ASM contains a Device Control Aggregate and Device Control Documents to capture parameters used in the data acquisiton.

No such parameters are included in the Cedex BioHT export, and those only device type of “cell culture analyzer” needs to be recorded.

Analyte documents within the Analyte Aggregate Document should be return in the alphabetical order of the analyte name.

In the event that the measurement is invalid such has when a measurement is below the detection range - as indicate by a < TEST RNG entry within the Flag column (tenth column - including blank columns), then a null value should be returned to the analyte document.

There may be cases where multiple tests for the sample analyte type were performed for the same sample (i.e. Order ID), this is indicated with entries with different test codes. For example, a single Order ID having entries for TP2B, TP2D, and TP2LB - all of which map to a Total Protein measurement. In these cases it is expected that only one entry will return a valid measurement, in which case this value should be recorded within the measurement document. In the event that there are multiple test codes mapping to the same analyte, then the additional results should be placed in a separate measurement document within the same measurement aggregate document and cell culture analyzer document, with the sample information repeated in the accompanying sample document.

Other properties within Measurement Document

Note, the properties of ‘optical density’ and ‘osmolality’ should be recorded within the measurement document and not individual analyte documents as these reflect properties of the sample as whole and do not pertain to individual analytes.
