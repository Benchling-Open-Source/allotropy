Electrophoresis Aggregate Document

The electrophoresis aggregate document represents the highest level of organization in the capillary electrophoresis ASM, and will contain the device system document - for recording details about the instrument, the data system document - for recording details about the data file, the electrophoresis document - an array for recording data from each sample, and where appropriate a calculated data aggregate document for recording calculated results derived from multiple electrophoresis documents (e.g. aggregated results from multiple measurements and/or samples).

Electrophoresis Document

The electrophoresis document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the electrophoresis technique is performed. In this framework the electrophoresis  technique is performed for each sample processed (rather than plate or cassette) in the instrument software export. This means that information from each sample should be recorded in a separate electrophoresis document.

Enumerated list (see below)

Sample role type enumerated list (see resource here for other ontological definitions):

  • control sample role - A control sample role is a sample role of a material sample that does contain a known amount of analyte for control purposes.

  • standard sample role - A standard sample role is a sample role of a material sample that does contain a known amount of standard.

  • validation sample role - A validation sample role is a sample role of a material sample that is used for validation purposes.

  • experiment sample role - An experiment sample is a role that is played by a material object of interest in an assay used to obtain generalizable information about the sample source.

  • sample role - A sample role type is a classification datum that classifies samples by the type of sample role in the experiment

  • spiked sample role - A spiked sample role is a sample role of a material sample that does contain a spike substance.

  • blank role - A blank role is a sample role of a material that does not contain any analyte.

  • unknown sample role - An unknown sample role is a sample role of a material of an unknown substance, or that does contain an unknown amount of analyte.

  • calibration sample role - A calibration sample role is a sample role of a material that does contain a known amount of substance for calibration purposes.

  • unspiked sample role - An unspiked sample role is a sample role of a material sample that does not contain a spike substance.

  • specimen role - Specimen role is the sample role of a prepared fraction of a portion of material that is analyzed in a particular experiment.

  • quality control sample role - A quality control sample role is a sample role of a material sample that is used for purposes of quality control.

  • reference sample role - A reference sample role is a sample role of a material sample that does contain a known amount of analyte.

  • positive control sample role - A positive control sample role is a control sample role intended to produce a positive result if the test is working as expected.
  • negative control sample role - A negative control sample role is a control sample role intended to produce a negative result, to help correlate a positive result with the variable being tested.

Electropherogram Data Cube

The electropherogram data cube contains details of the electrophoresis measurement of fluorescence/absorbance versus elapsed time. If exporting the raw data, then the processed data would be relative to the reference sample (e.g., ladder). On most instruments, it is possible to export the data with it relative to the ladder). This exported data would be used to generate the histogram to visualize the peaks and generate the processed data.

Processed Data Aggregate Document / Processed Data Document

The processed data document contains records of information derived from the measurement data itself. For electrophoresis experiments the reporting of features of the electropherogram such as peaks, regions, or translations of the electropherogram from ‘time’ to ‘molecular mass’ or ‘polymer length’ (e.g. nucleotide length or base-pair length) are structured as  processed data. The processed data document is captured within a processed data aggregate document, to allow for recording of multiple processed data documents.

Peak List / Peak

The peak records the details of peaks identified within the electropherogram. The peak is captured within a peak list, to allow for recording the details of multiple peaks.

Note: the peak and peak list are equivalent to a ‘peak document’ and a ‘peak aggregate document’, however, the peak and peak list terms are legacy terms in the Allotrope Foundation Ontology predating the introduction of the ‘document’ terms.

Error Aggregate Document / Error Document

The error document within the ASM is intended to record information about any errors returned by the instrument in the course of performing the measurement. As multiple errors may be reported per measurement, these are structured within an error aggregate document which in turn contains the error document. The error document is an array of objects, each recording details about a specific error.

Calculated Data Aggregate Document / Calculated Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document contains information derived from a single measurement, while the calculated data document is used to report the output of calculations which uses as inputs multiple measurements from the same or different samples within the experiment.

Note to Author: Examples of data that would recorded within calculated data document could include: background subtracted measurements where the value from a separate “Blank” sample is subtracted from the measure; the average of measurements from technical replicates within the experiment; or the concentration of a sample derived from comparing the measured value to a standard curve generated from other measures within with the experiment. 

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the file export.

All calculated data documents should be placed within the calculated data aggregate document within the electrophoresis aggregate document.

Data Source Aggregate Document

Within the ASM, the data source data document is used to reference other information with  the ASM file which serves as the input to the calculation reported within the calculated data document. As a calculation may have multiple inputs, these are structured within a data source aggregate document which in turn contains the data source document. The data source document is an array of objects, each recording details about a data that served as input to the calculation.
