Multi Analyte Profiling Document

The multi analyte profiling document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the multi-analyte-profiling technique is performed. In this framework the multi-analyte-profiling is performed for each sample processed by the instrument. As multi-analyte-profiling analyzers are usually plate-based instruments, each well of the plate should be recorded in a separate multi analyte profiling document.

Measurement Document

The measurement document is an array of objects, each containing details about individual measurements recorded from each well within the instrument software export.

Note to Author: Multi-analyte-profiling is a ‘multiplexed’ assay in which data about multiple analytes are reported from a single sample measured by the instrument using a single set of instrument settings.

The pattern for modeling this data is to use a single measurement document in which the data for each analyte is reported in an analyte document - which is a child document within the measurement document. As opposed to the data from each analyte being modeled as a ‘measurement’

IF a sample were to be processed using different instrument settings (not common) than each instance (and the associated analyte data) would be modeled in separate measurement documents.

Sample role type enumerated list (see resource here for other ontological definitions):

  • control sample role - A control sample role is a sample role of a material sample that does contain a known amount of analyte for control purposes.

  • standard sample role - A standard sample role is a sample role of a material sample that does contain a known amount of standard.

  • validation sample role - A validation sample role is a sample role of a material sample that is used for validation purposes.

  • experiment sample role - An experiment sample is a role that is played by a material object of interest in an assay used to obtain generalizable information about the sample source.

  • sample role - A sample role type is a classification datum that classifies samples by the type of sample role in the experiment

  • spiked sample role - A spiked sample role is a sample role of a material sample that does contain a spike substance.

  • blank role - A blank role is a sample role of a material that does not contain any analyte.

  • unknown sample role - An unknown sample role is a sample role of a material of an unknown substance, or that does contain an unknown amount of analyte.

  • calibration sample role - A calibration sample role is a sample role of a material that does contain a known amount of substance for calibration purposes.

  • unspiked sample role - An unspiked sample role is a sample role of a material sample that does not contain a spike substance.

  • specimen role - Specimen role is the sample role of a prepared fraction of a portion of material that is analyzed in a particular experiment.

  • quality control sample role - A quality control sample role is a sample role of a material sample that is used for purposes of quality control.

  • reference sample role - A reference sample role is a sample role of a material sample that does contain a known amount of analyte.

    Benchling Proposed - https://gitlab.com/allotrope/afo/-/issues/1041

  • positive control sample role - A positive control sample role is a control sample role intended to represent the expected experimental phenotype
  • negative control sample role - A negative control sample role is a control sample role intended to represent the opposite of the experimental phenotype
  • heterozygous control sample role - a heterozygous control sample role is a control sample known to be heterozygous for a gene
  • homozygous control sample role - a homozygous control sample role is a control sample known to be homozygous for a gene
  • reference DNA control sample role - a reference DNA control sample role is a control sample containing a known reference DNA
  • DNA amplification control sample role- a DNA amplification control sample role is a control sample for interpretation of the results of DNA amplification.

Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.

Error Aggregate Document / Error Document

The error document within the ASM is intended to record information about any errors returned by the instrument in the course of performing the measurement. Calculated Data Aggregate Document / Calculated Document

Within the ASM, the calculated data document is used to record results that are derived from calculations performed using data across different measurement documents and/or other calculated data documents . A calculated data document differs from a processed data document in that a processed data document contains information derived from a single measurement, while the calculated data document is used to report the output of calculations which uses as inputs multiple measurements from the same or different samples within the experiment.

Note to Author: Examples of data that would recorded within calculated data document could include: background subtracted measurements where the value from a separate “Blank” sample is subtracted from the measur e; the average of measurements from technical replicates within the experiment; or the concentration of a sample derived from comparing the measured value to a standard curve generated from other measures within with the experiment. 

While a calculated data document records calculated results, the adapter is not expected to perform any calculations, and rather parse calculated values which appear within the file export.

All calculated data documents should be placed within the calculated data aggregate document within the multi analyte profiling aggregate document.

Within the ASM, the data source data document is used to reference other information with  the ASM file which serves as the input to the calculation reported within the calculated data document. As a calculation may have multiple inputs, these are structured within a data source aggregate document which in turn contains the data source document. The data source document is an array of objects, each recording details about a data that served as input to the calculation.
