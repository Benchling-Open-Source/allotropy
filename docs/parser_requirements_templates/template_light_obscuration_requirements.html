Light Obscuration Aggregate Document

The light obscuration aggregate document represents the highest level of organization in the Light Obscuration ASM, and will contain the device system document - for recording details about the instrument, the data system document - for recording details about the data file, the light obscuration document - for recording data from each sample.

Light Obscuration Document

The light obscuration document is an example of a technique document in the context of the Allotrope Simple Model, and is meant to represent each time the light obscuration technique is performed. In this framework the light obscuration technique is performed for each sample in the instrument software export. This means that information from each sample should be recorded in a separate light obscuration document.

Measurement Aggregate Document

The measurement document is an array of objects, each containing details about individual measurements recorded from each well within the instrument software export. Device Control Aggregate Document / Device Control Document

The device control document within the ASM is intended to record information about device settings under which each measurement was performed. The measurement document will always contain a device control aggregate document which in turn contains device control document. The device control document is an array of objects, each recording details about a component of the instrument.

Because the same device settings are typically used for the measurement of all samples within the dataset, it is expected that these values will be repeated across multiple measurement documents with the ASM file.
